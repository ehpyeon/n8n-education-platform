데이터의 두 가지 종류

정보를 정리하는 방식에 따라 데이터는 크게 두 종류로 나뉩니다
📊 구조화된 데이터
  • 미리 정해진 형식에 맞춰 정리된 정보
  • 행과 열로 구성된 표 형태
  • 검색과 분석이 쉬움
  • Excel 시트처럼 규칙적인 구조

📄 비구조화된 데이터

📝 핵심 포인트: 구조화된 데이터는 정리된 서랍장처럼 찾기 쉽고, 비구조화된 데이터는 일기장처럼 내용은 풍부하지만 특정 정보를 찾기 어렵습니다.

일상 생활 비유로 이해하기

데이터 저장 방식을 집안 정리에 비유해서 쉽게 이해해봅시다
🗂️ 구조화된 데이터 = 체계적인 서랍장
  • 각 서랍에 정해진 물건만 보관
  • 양말은 양말 서랍에, 속옷은 속옷 서랍에
  • 라벨이 붙어있어 쉽게 찾을 수 있음
  • 새로운 물건도 규칙에 맞춰 정리
  • 정리는 번거롭지만 찾기는 쉬움
📦 비구조화된 데이터 = 큰 박스
  • 모든 물건을 한 박스에 보관
  • 양말, 속옷, 문서, 사진 모두 섞여있음
  • 다양한 물건을 자유롭게 넣을 수 있음
  • 특정 물건을 찾으려면 전체를 뒤져야 함
  • 정리는 쉽지만 찾기는 어려움
구분 구조화된 데이터 비구조화된 데이터

데이터베이스란 무엇인가?

데이터베이스는 정보를 체계적으로 보관하고 관리하는 전용 창고입니다
🏢 데이터베이스의 역할
  1. 정보 저장 - 데이터를 안전하게 보관
  2. 정보 검색 - 원하는 데이터를 빠르게 찾기
  3. 정보 수정 - 데이터를 업데이트하고 관리
  4. 정보 보안 - 권한에 따라 접근 제어
💼 비즈니스에서의 활용
  • 고객 정보 관리 (이름, 연락처, 구매내역)
  • 주문 처리 시스템 (주문번호, 상품, 배송)
  • 직원 관리 (인사정보, 급여, 근태)
  • 재고 관리 (상품코드, 수량, 가격)
🎯 왜 필요한가?
문제점 해결책
Excel 파일이 너무 커짐 대용량 데이터 처리
여러 명이 동시 작업 불가 동시 접근 지원
데이터 백업과 복구 어려움 자동 백업 시스템
보안 관리 한계 사용자별 권한 관리

n8n은 다양한 데이터베이스를 노드로 쉽게 연결할 수 있도록 지원합니다
📋 지원하는 데이터베이스 노드
노드명 데이터베이스 난이도

Airtable 에어테이블 ⭐⭐
Postgres PostgreSQL ⭐⭐⭐
MySQL MySQL ⭐⭐⭐
MongoDB 몽고DB ⭐⭐⭐
🔄 공통 작업 타입
  1. Create (생성) - 새로운 데이터 추가
  2. Read (읽기) - 기존 데이터 조회
  3. Update (수정) - 데이터 내용 변경
  4. Delete (삭제) - 불필요한 데이터 제거
이를 CRUD 작업이라고 부르며, 모든 데이터베이스 노드에서 동일하게 지원합니다.
🎯 연결 설정 공통 단계
  1. 노드 추가 - 원하는 데이터베이스 노드를 워크플로우에 추가
  2. 크레덴셜 설정 - 데이터베이스 접속 정보 입력 (API 키, 계정 정보)
  3. 작업 유형 선택 - Create, Read, Update, Delete 중 선택
  4. 테이블/시트 지정 - 작업할 데이터의 위치 설정
  5. 데이터 매핑 - 입력 데이터와 데이터베이스 필드 연결

📊 Google Sheets 연결 실습

가장 쉬운 Google Sheets 노드를 사용해서 실제 연결 방법을 배워봅시다
1️⃣ 크레덴셜 설정
  1. Google Sheets 노드 추가
    • 노드 팔레트에서 "Google Sheets" 검색
    • 워크플로우에 드래그 앤 드롭
  2. 크레덴셜 생성
    • "Credential to connect with" 드롭다운 클릭
    • "Create New" 선택
    • Google 계정으로 로그인 인증
2️⃣ 스프레드시트 설정
  1. 작업 유형 선택
    • Operation: "Append" (데이터 추가)
    • 다른 옵션: Read, Update, Delete
  2. 시트 정보 입력
    • Document ID: 구글 시트 URL의 ID 부분
    • Sheet: 작업할 시트명 (예: Sheet1)
    • Range: 데이터 범위 (예: A:C)
🔧 실제 설정 예시
// Google Sheets URL 예시 https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms/edit // Document ID는 URL 중간의 긴 문자열 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms // 설정 예시 - Operation: Append - Document ID: 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms - Sheet: 고객명단 - Range: A:D
💡 팁: Document ID는 Google Sheets URL을 복사해서 중간의 긴 영숫자 조합을 찾으면 됩니다. 보통 '/d/' 뒤와 '/edit' 앞 부분입니다.

🎨 Airtable 연결 실습

Airtable은 API 키를 사용하여 더 세밀한 권한 관리가 가능합니다
1️⃣ API 키 발급받기
  1. Airtable 계정 설정
    • airtable.com 로그인
    • 우상단 프로필 → "Developer hub"
    • "Personal access tokens" 선택
    • "Create new token" 클릭
  2. 권한 설정
    • Token name: "n8n 연동" 등 식별 가능한 이름
    • Scopes: data.records:read, data.records:write
    • Access: 사용할 베이스 선택
2️⃣ n8n에서 연결 설정
  1. Airtable 노드 추가
    • "Airtable" 노드 워크플로우에 추가
    • 크레덴셜 "Create New" 선택
  2. 연결 정보 입력
    • API Key: 방금 생성한 토큰 입력
    • Base ID: Airtable 베이스의 고유 ID
    • Table: 작업할 테이블명
🔍 Base ID 찾는 방법

방법 1: URL에서 확인

// Airtable URL 예시 https://airtable.com/app12345678901234/tblABCDEFGHIJKLMN // Base ID는 app으로 시작하는 부분 app12345678901234

방법 2: API 문서에서 확인

  1. 베이스 페이지에서 "Help" 메뉴
  2. "API documentation" 선택
  3. 상단에 Base ID 표시됨
⚠️ 주의사항: API 키는 계정의 모든 권한을 가지므로 안전하게 보관해야 합니다. n8n의 크레덴셜 기능을 사용하면 암호화되어 저장됩니다.

웹사이트 문의 양식을 통해 들어온 고객 정보를 자동으로 데이터베이스에 저장하는 시스템을 만들어봅시다

항목 내용

실전 시나리오 2: 프로젝트 관리 시스템

다양한 뷰와 관계형 데이터가 필요한 프로젝트 관리를 위해 Airtable을 활용해봅시다
📊 프로젝트 데이터 구조
테이블 주요 필드
Projects 프로젝트명, 시작일, 마감일, 상태
Tasks 작업명, 담당자, 우선순위, 진행률
Team Members 이름, 역할, 이메일, 연락처
Time Tracking 날짜, 소요시간, 작업내용
🎨 Airtable의 장점 활용
  • 칸반 뷰 - 작업 진행 상황 시각화
  • 캘린더 뷰 - 마감일 일정 관리
  • 갤러리 뷰 - 프로젝트 포트폴리오
  • 연결 필드 - 프로젝트↔작업↔팀원 관계
  • 공식 필드 - 진행률 자동 계산
  • 조건부 서식 - 마감 임박 작업 강조
🔄 n8n 자동화 워크플로우

📅 매주 리포트

  1. Cron 트리거 (매주 월요일)

  3. 리포트 형식으로 데이터 가공
  4. 팀 이메일로 전송

⚠️ 마감일 알림

  4. 개별 알림 메시지 발송

📊 대시보드 업데이트

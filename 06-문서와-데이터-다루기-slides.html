⚙️ n8n 파일 처리 방법

n8n이 파일을 다루는 두 가지 똑똑한 방법
📁 파일 직접 전달 (Binary Data)
n8n에서는 파일 자체를 binary data로 노드 간 전달합니다
  • 앞 노드에서 받은 파일을 그대로 사용
  • 연결된 노드에서 binary 부분의 명칭 입력
  • 파일 내용을 메모리에서 직접 처리
🌐 URL 전달 후 다운로드
파일 위치(URL)만 전달하고 필요할 때 HTTP Request로 다운로드
  • URL을 받아 뒤에서 다운받아 사용
  • HTTP Request 노드로 파일 다운로드
  • 메모리 절약하며 큰 파일도 효율적 처리

💾 Binary Data를 이용한 직접 파일 처리

컴퓨터에서 파일을 직접 가져와 도서관 사서처럼 바로 분석
📁 파일 업로드 트리거 설정
  1. n8n 워크플로우 생성
    • 새 워크플로우 만들기
    • Manual Trigger 노드 추가
    • 파일 업로드 기능 활성화
  2. 업로드 설정 구성
    • 허용 파일 형식: PDF, DOCX, TXT
    • 최대 파일 크기 제한
    • Binary Data 자동 처리
🤖 Upstage Document Parse 연결
  1. Document Parse 노드 추가
    • Upstage Document Parse 선택
    • 이전 노드에서 Binary Data 받기
    • 파일 입력 방식: Binary 선택
  2. 파싱 결과 확인
    • 텍스트 추출 결과 검토
    • 구조화된 데이터 형태 확인
    • 다음 단계로 데이터 전달
💡 파일을 직접 드래그&드롭하듯 쉽게 처리하는 가장 직관적인 방법

📝 Google Form을 통한 파일 URL 수집

고객이 온라인 주문서에 파일 위치를 적어주는 시스템
📋 Google Form 설정
  1. 새 Google Form 생성
    • forms.google.com 접속
    • "문서 URL 제출" 제목 설정
    • 설명: 분석할 파일 링크 입력
  2. 질문 항목 구성
    • 파일 URL 입력란 (필수)
    • 파일 유형 선택 (PDF/DOCX)
    • 요청자 이메일 (선택)
  3. 응답 설정
    • 응답 후 확인 메시지
    • 스프레드시트 자동 생성
    • 실시간 알림 설정
🔗 n8n Webhook 연결
  1. Webhook 트리거 설정
    • n8n에서 Webhook 노드 추가
    • GET/POST 방식 선택
    • 고유 Webhook URL 생성
  2. Google Form 연동
    • Form 응답 시 Webhook 호출
    • URL 데이터 자동 전달
    • JSON 형태로 데이터 수신
🏢 24시간 열린 온라인 접수창구처럼 언제든 파일 요청 가능

🌐 HTTP Request로 파일 다운로드 후 처리

택배 기사가 주소를 받고 물건을 가져다주는 것처럼 URL로 파일 수령
📥 HTTP Request 설정
  1. HTTP Request 노드 추가
    • Request Method: GET
    • URL: Google Form에서 받은 링크
    • Response Format: Binary
  2. 다운로드 옵션 설정
    • Timeout: 30초 설정
    • Follow Redirects: 활성화
    • Binary Response: 체크
  3. 에러 처리
    • 404 에러 시 대체 로직
    • 권한 없음 시 알림
    • 재시도 로직 구성

🏭 데이터베이스 - 최첨단 물류 창고

단순히 물건을 쌓아놓는 창고가 아닌, 어디에 무엇이 있는지 정확히 알고 바로 찾아주는 똑똑한 창고
📊 일반 창고 vs 똑똑한 창고
구분 엑셀/CSV (일반) 데이터베이스 (똑똑)
중복 보관 같은 물건 여러 곳에 한 곳에만 보관
동시 사용 한 명만 이용 가능 여러 명이 동시에
물건 찾기 직접 돌아다니며 시스템이 자동으로
보안 관리 창고 문 잠금만 구역별 출입 통제
🗂️ 창고 배치도 (스키마)
새 집으로 이사할 때 어느 방에 뭘 둘지 미리 계획하는 것처럼
고객 서랍장 (customers): ├── 고객번호 (id) - 1, 2, 3... 자동 ├── 고객 이름 (name) - "김철수" ├── 연락처 (phone) - "010-1234-5678" ├── 이메일 (email) - "kim@email.com" └── 첫 문의일 (created_at) - 자동으로 오늘 날짜 문의 서랍장 (inquiries): ├── 문의번호 (id) - 1, 2, 3... 자동 ├── 고객번호 (customer_id) │ - 위 고객과 연결 ├── 문의내용 (content) │ - "배송이 언제 되나요?" ├── 담당자 (assignee) - "이영희" ├── 상태 (status) │ - "접수중", "처리중", "완료" └── 접수일 (created_at) - 자동으로 오늘 날짜

🏪 Supabase - 동네 친절한 창고

"어서오세요! 바로 써보세요!" 복잡한 설정 없이 쉽게 시작하는 초보자 창고
✨ 동네 Supabase 창고의 장점
장점 일반 창고 Supabase
비용 비싼 보증금 + 월세 무료로 시작!
설정 복잡한 계약과 서류 클릭 몇 번이면 끝
편의성 직접 키 관리, 보안 자동으로 다 해줌
연결 직접 트럭 대절 n8n이 자동 연결
📋 창고 계약 4단계
  1. 창고 계약 (회원가입)
    supabase.com → "무료로 시작하기"
  2. 내 창고 이름 정하기
    "New Project" → 창고 이름 설정
  3. 창고 비밀번호 설정
    나만 아는 마스터키 생성
  4. 3-5분 기다리면 완성!
    내 전용 창고 준비 완료
🎯 5분이면 전문가급 데이터베이스 완성!

🗂️ 창고 안에 서랍장 만들기

고객 문의 시스템용 깔끔한 서랍장 설계하기
👥 고객정보 서랍장 설계
고객정보 서랍장 (customers): ├── 번호표 (id) - 1, 2, 3... 자동 ├── 이름 (name) - "김철수" ├── 이메일 (email) - "kim@email.com" │ (중복 안됨 설정) ├── 연락처 (phone) - "010-1234-5678" └── 첫 문의일 (created_at) - 자동으로 오늘 날짜
🔗 Supabase에서 만들기:
  1. Table Editor → "New Table"
  2. 테이블 이름: "customers"
  3. 컬럼 추가 및 타입 설정
  4. 제약조건 설정 (이메일 유니크)
📋 문의내역 서랍장 설계
문의내역 서랍장 (inquiries): ├── 문의번호 (id) - 1, 2, 3... 자동 ├── 고객번호 (customer_id) │ - 위 고객과 연결 (Foreign Key) ├── 제목 (title) - "배송 문의" ├── 문의내용 (content) │ - "배송이 언제 되나요?" ├── 담당자 (assignee) - "이영희" ├── 상태 (status) │ - "접수중", "처리중", "완료" ├── 우선순위 (priority) - "높음", "보통", "낮음" └── 접수일 (created_at) - 자동으로 오늘 날짜
🔗 관계 설정:
  • customer_id → customers.id 연결
  • 한 고객이 여러 문의 가능
  • 고객 정보 자동 연동

🔗 n8n과 Supabase 창고 연결하기

택배기사에게 우리 집 비밀번호를 알려주듯 n8n에게 창고 열쇠 주기
🔑 1단계: 창고 열쇠 구하기
  1. Supabase에서 API 키 확인
    • Settings → API 메뉴
    • Project URL 복사
    • service_role Key 복사
  2. n8n에서 크레덴셜 생성
    • Credentials → "Add Credential"
    • "Supabase API" 선택
    • URL과 Key 입력
⚙️ 2단계: 데이터 저장 노드 설정
Supabase 노드 설정: Operation: Insert (데이터 추가) Table: customers (테이블 선택) 데이터 매핑: - name → {{$json.고객이름}} - email → {{$json.고객이메일}} - phone → {{$json.연락처}} 자동 입력: - id: 자동 생성 - created_at: 현재 시간
💡 실제 워크플로우:
  • Webhook → Set(데이터 정리) → Supabase(저장)
  • 조건 분기로 중복 고객 체크
  • 에러 처리로 안전한 저장

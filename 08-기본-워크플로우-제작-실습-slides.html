🏗️ 기본 자동화 워크플로우 구축

요리 레시피처럼 단계별로 따라하는 첫 번째 자동화 시스템

  1. 간단한 고객 응답 자동화 만들기 - Form에서 접수부터 자동 응답까지
  2. 워크플로우 설계의 기본 원칙 - 실패하지 않는 자동화를 위한 필수 규칙
  3. 테스트와 디버깅 방법 - 문제 발생 시 빠르게 해결하는 방법
  4. 성능 최적화 기본기 - 느려지지 않고 안정적으로 돌아가는 시스템

🎯 우리가 만들 첫 번째 자동화 시스템

고객 문의 접수부터 자동 응답까지, 완전히 사람 손을 거치지 않는 시스템을 만들어보겠습니다
📝 시스템 개요
  • 접수: 웹 폼으로 고객 문의 받기
  • 저장: 문의 내용을 체계적으로 보관
  • 분류: 문의 유형을 자동으로 판단
  • 응답: 적절한 답변 자동 발송
  • 알림: 담당자에게 처리 현황 통보
🍳 요리 레시피로 비유하면
  1. 재료 준비 (Form Trigger)
    • 고객 문의라는 '재료' 받기
  2. 손질하기 (Set 노드)
    • 받은 정보를 정리하고 분류
  3. 조리하기 (IF 노드)
    • 문의 유형에 따라 다르게 처리
  4. 서빙하기 (Gmail 노드)
    • 완성된 답변을 고객에게 전달

📋 고객 문의 접수 창구 만들기

Form Trigger로 고객이 언제든지 문의할 수 있는 온라인 창구를 설치합니다
🎪 Form Trigger 설정
  1. Trigger 노드 추가
    • 왼쪽 패널에서 'Form Trigger' 검색
    • 캔버스에 드래그앤드롭으로 배치
  2. 폼 필드 구성
    • 이름 (Name): Text 타입
    • 이메일 (Email): Email 타입
    • 문의 유형 (Type): Select 타입
    • 문의 내용 (Message): Textarea 타입
  3. 폼 URL 확인
    • 생성된 URL을 복사해서 테스트
    • 실제로 문의 작성해보기
📞 문의 유형 분류
문의 유형 설명 자동 응답
일반 문의 제품/서비스 관련 질문 상세 안내 발송
기술 지원 사용법이나 오류 해결 기술팀 연결 안내
불만 접수 서비스 개선 요청 진심 사과 및 처리 절차
제휴 제안 비즈니스 협력 요청 담당팀 연결 안내

📦 받은 정보 깔끔하게 정리하기

Set 노드로 폼에서 받은 정보를 우리가 사용하기 편한 형태로 정리합니다
⚙️ Set 노드 설정
  1. Set 노드 추가
    • Form Trigger 다음에 Set 노드 연결
    • 'Manual' 모드 선택
  2. 필드 정리
    • customer_name: {{ $json.name }}
    • customer_email: {{ $json.email }}
    • inquiry_type: {{ $json.type }}
    • inquiry_message: {{ $json.message }}
    • received_time: {{ $now }}
    • status: "접수완료"
💾 Google Sheets에 저장
  1. Google Sheets 노드 추가
    • Set 노드 다음에 연결
    • Operation: 'Append' 선택
  2. 스프레드시트 연결
    • 미리 만든 '고객문의' 시트 연결
    • 헤더: 이름, 이메일, 유형, 내용, 접수시간, 상태
  3. 데이터 매핑
    • 각 열에 Set 노드 결과값 연결
💡 꿀팁: Google Sheets는 간단한 데이터베이스 역할을 해주어 추후 분석이나 관리에 매우 유용합니다.

🔀 문의 유형별로 다른 처리하기

IF 노드로 문의 유형에 따라 서로 다른 자동 응답을 보내도록 설정합니다
🤔 IF 노드 조건 설정
  1. IF 노드 추가
    • Google Sheets 노드 다음에 연결
    • 'Simple' 모드 선택
  2. 조건 1: 일반 문의
    • Value 1: {{ $json.inquiry_type }}
    • Operation: 'equal'
    • Value 2: "일반 문의"
  3. 추가 조건 설정
    • 기술 지원, 불만 접수, 제휴 제안 조건 추가
    • 각 조건별로 다른 경로 생성
📧 응답 메일 템플릿
안녕하세요, {{ $json.customer_name }}님 {{ $json.inquiry_type }} 문의를 접수했습니다. [일반 문의인 경우] 상세한 안내를 위해 담당자가 24시간 내에 연락드리겠습니다. [기술 지원인 경우] 기술팀에서 문제를 검토 중입니다. 해결 방안을 찾는 대로 연락드리겠습니다. [불만 접수인 경우] 불편을 끼쳐드려 진심으로 사과드립니다. 즉시 개선 작업에 착수하겠습니다. 감사합니다.

📨 자동 응답 이메일 발송하기

Gmail 노드로 문의 유형별 맞춤형 응답을 자동으로 발송합니다
✉️ Gmail 노드 설정
  1. Gmail 노드 추가
    • 각 IF 분기별로 Gmail 노드 연결
    • Operation: 'Send' 선택
  2. 수신자 설정
    • To: {{ $json.customer_email }}
    • From: 우리 회사 이메일
  3. 메일 내용
    • Subject: [{{ $json.inquiry_type }}] 문의 접수 확인
    • Body: 유형별 맞춤 템플릿 사용
📱 Slack 알림 설정
  1. Slack 노드 추가
    • Gmail 노드와 병렬로 연결
    • 담당팀 채널에 알림 발송
  2. 알림 메시지 구성
    • 🔔 새 문의: {{ $json.inquiry_type }}
    • 고객: {{ $json.customer_name }}
    • 내용: {{ $json.inquiry_message }}
    • 접수시간: {{ $json.received_time }}
📊 중요한 포인트: 고객에게는 즉시 응답하고, 내부팀에는 처리 알림을 보내 누구도 놓치지 않게 합니다.

🛡️ 실패하지 않는 자동화 만들기

예상치 못한 상황이 발생해도 시스템이 멈추지 않도록 방어막을 설치합니다
⚠️ 주요 에러 상황들
  • 네트워크 연결 실패: 인터넷이 불안정한 경우
  • Gmail 발송 실패: 계정 문제나 용량 초과
  • Sheets 저장 실패: 권한 문제나 시트 오류
  • 잘못된 데이터: 이메일 주소 오타 등
🔧 Try/Catch 에러 처리
  1. 핵심 노드들을 Try 블록으로 감싸기
    • Gmail 노드 주변에 Try/Catch 설치
    • Google Sheets 노드도 보호
  2. Catch에서 대체 동작 설정
    • 메일 실패 시 → 텍스트 파일로 로그 저장
    • Slack 알림으로 관리자에게 통보
✅ 데이터 검증 체크리스트
  1. 입력 데이터 검증
    • 이메일 주소 형식 확인: 정규표현식 사용
    • 필수 필드 누락 검사: 이름, 이메일, 내용
    • 문의 유형 값 유효성 검증
  2. 처리 결과 확인
    • 메일 발송 성공 상태 체크
    • 데이터베이스 저장 완료 확인
    • 각 단계별 처리 시간 기록

🔍 워크플로우 테스트 및 문제 해결

만든 자동화가 제대로 작동하는지 체계적으로 테스트하고 문제를 해결합니다
🎯 단계별 테스트 방법
  1. 개별 노드 테스트
    • 각 노드를 하나씩 실행해보기
    • 데이터가 제대로 전달되는지 확인
    • 핀(Pin) 기능으로 중간 결과 고정
  2. 전체 워크플로우 테스트
    • Form에서 실제 문의 작성
    • 각 문의 유형별로 테스트
    • 응답 메일과 Slack 알림 확인
  3. 예외 상황 테스트
    • 잘못된 이메일 주소로 테스트
    • 빈 필드가 있는 상태로 테스트
    • 네트워크 연결 불안정한 상황 시뮬레이션
🐛 문제 해결 가이드
문제 상황 확인 사항 해결 방법
워크플로우가 실행되지 않음 Trigger 설정 Form URL 재생성, 테스트 실행
메일이 발송되지 않음 Gmail 인증 크레덴셜 재설정, 권한 확인
데이터가 저장되지 않음 Sheets 연결 스프레드시트 권한, 시트명 확인
조건 분기가 작동하지 않음 IF 조건 설정 데이터 타입, 조건 값 정확성 확인

⚡ 빠르고 안정적인 자동화 만들기

많은 문의가 들어와도 느려지지 않고 안정적으로 처리되도록 시스템을 최적화합니다
🚀 성능 최적화 포인트
  • 병렬 처리: 메일 발송과 Slack 알림을 동시 실행
  • 데이터 최소화: 필요한 정보만 전달하기
  • 캐시 활용: 자주 사용하는 템플릿 미리 준비
  • 배치 크기 조절: 한 번에 처리할 건수 제한
📊 모니터링 설정
  1. 실행 로그 모니터링
    • Executions 탭에서 성공/실패 현황 확인
    • 처리 시간 모니터링
  2. 알림 설정
    • 에러 발생 시 즉시 Slack 알림
    • 일일 처리 건수 리포트

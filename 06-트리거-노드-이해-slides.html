🚀 트리거 노드 이해

자동화를 깨우는 다양한 신호들

  1. Manual Trigger - 수동 버튼 (테스트용)
  2. Schedule Trigger - 자동 알람시계 (정기 작업)
  3. Webhook - 실시간 벨 (즉시 반응)
  4. Email Trigger - 편지함 감시원 (이메일 기반)

🔔 Webhook - 똑똑한 초인종

누군가 버튼을 누르면 즉시 "띵동!" 하고 알려주는 인터넷 초인종
📮 일반 확인 vs Webhook
일반 방법 (폴링) Webhook 방법
5분마다 현관문 확인 택배기사가 벨 누름
계속 에너지 소모 필요할 때만 알림
늦게 발견 가능 즉시 알 수 있음
없어도 계속 확인 있을 때만 벨 울림
⚡ Webhook 장점
장점 비즈니스 효과
즉각성 1초 안에 담당자 알림
효율성 서버 자원 절약
정확성 문의 하나도 놓치지 않음
동시처리 여러 문의 동시 처리

🛠️ Webhook 실제 활용 사례

웹사이트 문의부터 주문 처리까지 즉시 자동화
🏃‍♂️ 고객 문의 즉시 대응 시스템
고객이 홈페이지에서 문의 작성 ↓ (즉시 Webhook 신호!) "새 문의 도착!" n8n이 바로 인식 ↓ 문의 내용을 업스테이지 AI가 분석 ("배송 문의네요!") ↓ 배송팀 이영희 과장에게 즉시 Slack 알림 ↓ 고객에게 "확인 중입니다" 자동 답장
🛒 온라인 주문
결제 시스템 → Webhook → 재고확인 → 포장팀 알림
👋 새 회원가입
가입 완료 → Webhook → 환영 이메일 → 쿠폰 발송
⭐ 제품 리뷰
리뷰 작성 → Webhook → 감정분석 → SNS 공유

⏰ 스케줄 트리거 - 정확한 알람시계

매일 오전 7시 알람처럼 정확한 시간에 자동 실행
📅 일상 vs 비즈니스 스케줄
일상 속 스케줄 비즈니스 스케줄
매일 오전 7시 알람 매일 오전 9시 매출 리포트
매주 일요일 쓰레기 배출 매주 월요일 고객 만족도 조사
매달 1일 월세 매월 1일 정산 보고서
1년에 한 번 건강검진 1년에 한 번 시스템 백업
⚙️ Cron 설정 방법
분 시 일 월 요일 * * * * * ↓ ↓ ↓ ↓ ↓ 0-59 0-23 1-31 1-12 0-6
* = "상관없음"

🕐 Cron 설정 실무 예시

우리 회사 업무시간에 맞춘 스케줄 설정하기
📋 자주 사용하는 패턴
원하는 시간 Cron 설정
평일 오전 9시 0 9 * * 1-5
매일 자정 0 0 * * *
매주 월요일 8시 0 8 * * 1
매월 1일 10시 0 10 1 * *
🏢 실제 업무 적용
매일 오전 9시: 어제 문의 요약 → 0 9 * * * 매주 월요일 8시: 주간 분석 → 0 8 * * 1 매월 1일 10시: 월간 통계 → 0 10 1 * *
⚡ 효율성 팁: 너무 자주 확인하면 서버 부담, 너무 가끔 확인하면 문제 발견 지연. 적당한 간격이 중요합니다!

🌏 한국 시간 맞추기 - 시차 주의!

n8n은 외국 시간(UTC)을 사용하므로 한국 시간 설정 시 주의 필요
🕐 시간 차이 계산
한국 시간 n8n 설정 시간
오전 9시 자정(0시)
오후 6시 오전 9시
자정(0시) 오후 3시
공식: 한국 시간 - 9시간 = n8n 설정
🔍 시간 확인 방법
  1. Set 노드 만들기
  2. {{ $now }} 입력해서 현재 시간 확인
  3. 한국 시간과 9시간 차이 확인
환경별 설정:
  • Docker: TZ=Asia/Seoul 환경변수
  • 서버: 시스템 시간대 한국 설정
  • 클라우드: n8n 타임존 변경

👆 메뉴얼 트리거 - 수동 스위치

사람의 판단이 필요한 중요한 업무에 사용하는 직접 실행 버튼
🤔 왜 수동으로?
일상 수동 버튼 비즈니스 수동 버튼
엘리베이터 (층 선택) 중요 이메일 발송

🛠️ 활용 사례
중요 공지사항 발송:
작성 → 검토 → 승인 버튼 → 전체 발송
시스템 백업:
점검 준비 → 상황 확인 → 백업 시작
긴급 복구:
문제 파악 → 복구 버튼 → 시스템 정상화

새 직원과 대화 연습하듯 AI 시스템 테스트

테스트 목적:
  • AI 답변 품질 확인

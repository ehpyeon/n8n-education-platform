⚙️ n8n 파일 처리 방법

n8n이 파일을 다루는 두 가지 똑똑한 방법
📁 파일 직접 전달 (Binary Data)
n8n에서는 파일 자체를 binary data로 노드 간 전달합니다
  • 앞 노드에서 받은 파일을 그대로 사용
  • 연결된 노드에서 binary 부분의 명칭 입력
  • 파일 내용을 메모리에서 직접 처리
🌐 URL 전달 후 다운로드
파일 위치(URL)만 전달하고 필요할 때 HTTP Request로 다운로드
  • URL을 받아 뒤에서 다운받아 사용
  • HTTP Request 노드로 파일 다운로드
  • 메모리 절약하며 큰 파일도 효율적 처리

💾 Binary Data를 이용한 직접 파일 처리

컴퓨터에서 파일을 직접 가져와 도서관 사서처럼 바로 분석
📁 파일 업로드 트리거 설정
  1. n8n 워크플로우 생성
    • 새 워크플로우 만들기
    • Manual Trigger 노드 추가
    • 파일 업로드 기능 활성화
  2. 업로드 설정 구성
    • 허용 파일 형식: PDF, DOCX, TXT
    • 최대 파일 크기 제한
    • Binary Data 자동 처리
🤖 Upstage Document Parse 연결
  1. Document Parse 노드 추가
    • Upstage Document Parse 선택
    • 이전 노드에서 Binary Data 받기
    • 파일 입력 방식: Binary 선택
  2. 파싱 결과 확인
    • 텍스트 추출 결과 검토
    • 구조화된 데이터 형태 확인
    • 다음 단계로 데이터 전달
💡 파일을 직접 드래그&드롭하듯 쉽게 처리하는 가장 직관적인 방법

📝 Google Form을 통한 파일 URL 수집

고객이 온라인 주문서에 파일 위치를 적어주는 시스템
📋 Google Form 설정
  1. 새 Google Form 생성
    • forms.google.com 접속
    • "문서 URL 제출" 제목 설정
    • 설명: 분석할 파일 링크 입력
  2. 질문 항목 구성
    • 파일 URL 입력란 (필수)
    • 파일 유형 선택 (PDF/DOCX)
    • 요청자 이메일 (선택)
  3. 응답 설정
    • 응답 후 확인 메시지
    • 스프레드시트 자동 생성
    • 실시간 알림 설정
🔗 n8n Webhook 연결
  1. Webhook 트리거 설정
    • n8n에서 Webhook 노드 추가
    • GET/POST 방식 선택
    • 고유 Webhook URL 생성
  2. Google Form 연동
    • Form 응답 시 Webhook 호출
    • URL 데이터 자동 전달
    • JSON 형태로 데이터 수신
🏢 24시간 열린 온라인 접수창구처럼 언제든 파일 요청 가능

🌐 HTTP Request로 파일 다운로드 후 처리

택배 기사가 주소를 받고 물건을 가져다주는 것처럼 URL로 파일 수령
📥 HTTP Request 설정
  1. HTTP Request 노드 추가
    • Request Method: GET
    • URL: Google Form에서 받은 링크
    • Response Format: Binary
  2. 다운로드 옵션 설정
    • Timeout: 30초 설정
    • Follow Redirects: 활성화
    • Binary Response: 체크
  3. 에러 처리
    • 404 에러 시 대체 로직
    • 권한 없음 시 알림
    • 재시도 로직 구성

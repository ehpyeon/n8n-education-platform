🏠 자동화 작업실 차리기 - 설치 방법 선택

새 사무실을 차릴 때 임대할지 구입할지 고민하듯, n8n도 설치 방법이 중요합니다

💻 로컬 설치 옵션

  • NPX - 가장 간단한 설치 방법
  • Docker - 컨테이너 기반 환경 독립성
  • 직접 서버 - 개인 서버나 VPS 설치

☁️ 클라우드 설치 옵션

  • n8n Cloud - 공식 클라우드 서비스 (유료)
  • AWS/GCP/Azure - 클라우드 플랫폼 배포
  • 서드파티 호스팅 - 관리형 서비스

🏆 풀옵션 월세 vs 자가 - 자체 호스팅 선택

집을 직접 소유하듯, 내 컴퓨터에서 n8n을 운영하면 완전한 자유도를 얻습니다
비교 요소 ☁️ 클라우드 호스팅 (풀옵션 월세) 💻 자체 호스팅 (자가)
💰 비용 매월 임대료 지속 지불 완전 무료 (전기료만)
🔒 데이터 보안 임대인이 관리하는 공간 내 집에서 완전 통제
🎨 커스터마이징 제한된 인테리어 변경 원하는 대로 모든 개조
📚 학습 효과 관리 경험 부족 집 관리 노하우 습득
🏠 결론: 내 집에서 살면 인테리어부터 관리까지 모든 것을 자유롭게 조절할 수 있습니다!

⚡ NPX를 이용한 간단 설치 (추천)

가장 빠르고 간단한 방법으로 n8n을 시작할 수 있습니다

📋 설치 전 준비사항

  1. Node.js 설치 확인
    node --version npm --version
  2. Node.js 미설치 시 다운로드
    • Windows/macOS: nodejs.org에서 LTS 버전 다운로드
    • Ubuntu/Debian: sudo apt install nodejs npm
    • 최소 요구사항: Node.js 18.x 이상

🚀 n8n 실행 과정

  1. n8n 실행
    npx n8n
  2. 브라우저에서 접속
    • 주소: http://localhost:5678
    • 초기 계정 설정 진행
    • 첫 워크플로우 생성 가능
🎉 완료! Node.js만 설치되어 있다면 몇 분만에 n8n이 실행되어 브라우저에서 확인할 수 있습니다.

🔐 관리자 설정 및 라이선스 활성화

설치 후 첫 접속시 어드민 계정을 생성하고 무료 라이선스를 받아 모든 기능을 활용해보세요

1️⃣ 초기 어드민 설정

이름, 이메일, 비밀번호 입력하여 관리자 계정 생성

2️⃣ 라이선스 키 요청

팝업에서 'Send me a free license key' 클릭하여 무료 라이선스 요청

3️⃣ 라이선스 키 입력

Settings > Usage and plan > Enter activation Key에 이메일로 받은 키 입력

🎉 완료! 이제 n8n의 모든 기능을 자유롭게 사용할 준비가 완료되었습니다!

🐳 Docker를 이용한 설치 (고급 사용자)

컨테이너 환경으로 더 안정적이고 격리된 설치를 원할 때 사용합니다

💻 터미널에서 설치

  1. Docker 설치 확인
    docker --version
  2. n8n Docker 이미지 실행 (임시)
    docker run -it --rm \ --name n8n \ -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n \ n8nio/n8n
  3. 데이터 영구 저장 설정 (권장)
    docker run -d \ --restart unless-stopped \ --name n8n \ -p 5678:5678 \ -v n8n_data:/home/node/.n8n \ n8nio/n8n

🖱️ Docker Desktop에서 설치

  1. Docker Hub에서 n8n 검색
  2. n8nio/n8n 이미지 Pull
  3. Run 실행 (포트 5678:5678 설정)
  4. localhost:5678 접속

🔧 자주 발생하는 설치 문제 해결

설치 과정에서 만날 수 있는 일반적인 문제들을 미리 대비합니다

🚫 포트 충돌 해결

# 포트 사용 확인 netstat -tulpn | grep 5678 # 다른 포트로 실행 npx n8n --port=8080

🔐 권한 문제 해결

# Node.js 전역 설치 권한 설정 sudo chown -R $(whoami) ~/.npm

💾 메모리 부족 문제

# Node.js 메모리 한도 증가 export NODE_OPTIONS="--max-old-space-size=4096"

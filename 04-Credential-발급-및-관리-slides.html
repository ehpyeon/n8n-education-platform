🔐 크레덴셜 발급 및 관리

외부와 소통하기 위한 신분증 만들기

  1. OpenAI API - AI 기능의 핵심 ($10 크레딧 권장)
  2. Gmail/Slack - 알림 및 소통 채널 연결
  3. Upstage API - 한국어 특화 AI 서비스
  4. 보안 원칙 - 최소 권한, 정기 교체, 안전한 보관

🗝️ 크레덴셜이 무엇인가요?

일상 생활 속 출입카드나 비밀번호처럼 외부 서비스 접근을 위한 인증 정보입니다
실생활 예시 n8n에서의 크레덴셜 역할
🏠 아파트 출입카드 Gmail 로그인 정보 내 Gmail에 접근해서 메일 읽기/보내기
🏦 은행 비밀번호 Google Sheets API 키 내 스프레드시트에서 데이터 가져오기/저장하기
🏢 회사 출입증 Slack 토큰 우리 팀 Slack에 메시지 보내기
📚 도서관 대출증 업스테이지 API 키 AI 서비스 사용하기

🛡️ 크레덴셜 보안의 중요성

집 열쇠를 잃어버리면 위험한 것처럼 크레덴셜도 안전하게 관리해야 합니다

⚠️ 안전한 관리 원칙

  • 절대 공유하지 않기
  • 개인 비밀번호처럼 혼자만 알고 있기
  • 정기적으로 확인하기
  • 이상한 활동이 없는지 가끔 체크하기
  • 필요한 권한만 주기
  • 꼭 필요한 기능에만 접근 허용하기

🔑 인증 방식의 종류와 특징

서비스마다 다른 인증 방법을 사용하므로 각각의 특징을 이해해야 합니다

🗝️ API Key 방식

가장 간단하고 널리 사용되는 인증 방식

특징:

  • 간단한 설정 - 키만 복사해서 붙여넣으면 즉시 사용
  • 영구적 사용 - 만료 기간이 없거나 매우 김
  • 전체 권한 - 해당 계정의 모든 권한에 접근
  • 보안 주의 - 키가 노출되면 모든 권한이 노출됨

사용 예시:

  • OpenAI API Key
  • Upstage API Key
  • 대부분의 AI 서비스 API

🔐 OAuth 토큰 방식

사용자를 대신하여 제한된 권한으로 서비스에 접근

특징:

  • 안전한 인증 - 실제 비밀번호를 공유하지 않음
  • 권한 제한 - 필요한 권한만 선택적으로 부여
  • 자동 갱신 - 토큰 만료 시 자동 새로운 토큰 발급
  • 사용자 제어 - 언제든 권한을 철회할 수 있음

사용 예시:

  • Google 서비스 (Gmail, Sheets, Drive)
  • 소셜 미디어 (Twitter, LinkedIn)
  • 클라우드 스토리지 (Dropbox, OneDrive)

🤖 OpenAI API 키 발급

ChatGPT 같은 AI 기능을 n8n에서 사용하기 위한 API 키를 발급받습니다
  1. 계정 생성
    • platform.openai.com 접속하여 계정 생성
  2. 결제 설정
    • Billing 섹션에서 결제 방법 등록
    • $10 정도 충전 권장 (소액으로 시작)
  3. API 키 생성
    • API Keys 메뉴에서 "Create new secret key" 클릭
  4. 키 복사 및 보관
    • 생성된 키를 즉시 복사 (한 번만 표시됨)
    • 안전한 곳에 보관
  5. n8n 등록
    • n8n에서 OpenAI 크레덴셜 생성 시 API 키 입력
⚠️ 중요 주의사항: API 키는 생성 시 한 번만 표시되므로 즉시 복사해야 하며, 사용량 모니터링으로 예상치 못한 과금을 방지하세요.

☁️ Google Cloud OAuth 등록

Gmail, Google Sheets, Drive 등 구글 서비스와 안전하게 연동합니다
  1. Google Cloud Console 접속
    • console.cloud.google.com 접속
  2. 프로젝트 생성
    • 새 프로젝트 생성 또는 기존 프로젝트 선택
  3. API 활성화
    • 필요한 API (Gmail, Sheets, Drive 등) 활성화
  4. OAuth 동의 화면 설정
    • 사용자 유형 선택 및 앱 정보 입력
  5. 사용자 인증 정보 생성
    • OAuth 2.0 클라이언트 ID 생성
    • n8n의 OAuth 콜백 URL 추가
  6. n8n에서 OAuth 설정
    • Client ID와 Client Secret을 n8n 크레덴셜에 입력
    • Connect 버튼으로 Google 계정 인증
    • 필요한 권한(Scope) 승인

🗄️ Supabase 프로젝트 설정

Firebase 대안인 Supabase로 데이터베이스와 인증 서비스를 연동합니다

🏗️ 프로젝트 생성

  1. 계정 생성
    • supabase.com에서 계정 생성
  2. 프로젝트 생성
    • "New project" 클릭하여 프로젝트 생성
  3. 데이터베이스 설정
    • 비밀번호 설정 및 리전 선택

🔑 API 키 수집

  • Project URL
  • https://your-project.supabase.co
  • Anon Key
  • 공개 읽기 작업용 키
  • Service Role Key
  • 서버 사이드 전체 권한 키

🔗 n8n 연동

  1. Supabase 크레덴셜 생성
  2. Project URL과 Service Role Key 입력
  3. 데이터베이스 연결 테스트

⚙️ 노드 기본 속성 설정

크레덴셜 외에도 각 노드의 다양한 속성값을 올바르게 설정해야 합니다

📋 필수 설정 항목

  • Credential - 해당 서비스의 인증 정보
  • Operation - 수행할 작업 유형 (Read, Write, Update 등)
  • Resource - 대상 리소스 (테이블, 파일, 메시지 등)

🔧 노드별 주요 설정 예시

HTTP Request 노드:

  • Method: GET, POST, PUT, DELETE
  • URL: 요청할 엔드포인트
  • Headers: 추가 헤더 정보
  • Body: 요청 데이터

OpenAI 노드:

  • Model: 사용할 AI 모델 선택
  • Prompt: 입력할 프롬프트
  • Max Tokens: 최대 응답 길이
  • Temperature: 창의성 수준 (0-1)

🔧 고급 옵션 설정

각 노드 하단의 Options 섹션에서 추가적인 세밀한 설정이 가능합니다

⚡ 공통 옵션

  • Timeout - 요청 제한 시간 설정
  • Retry - 실패 시 재시도 횟수
  • Rate Limit - 요청 간격 제한
  • Error Handling - 에러 발생 시 동작 방식

📊 데이터 처리 옵션

  • Response Format - 응답 데이터 형식 지정
  • Data Transformation - 데이터 변환 규칙
  • Filtering - 조건별 데이터 필터링
  • Pagination - 대용량 데이터 페이징 처리

📊 API 사용량 모니터링의 중요성

API 기반 서비스들은 사용량에 따른 과금이므로 지속적인 모니터링이 필수입니다

💰 비용 관리

모니터링 포인트:

  • 일일 사용량 - 예상치 못한 급증 패턴 감지
  • 월별 사용 추이 - 예산 대비 사용률 확인
  • 서비스별 비용 - 어떤 API가 가장 많이 사용되는지 파악
  • 에러율 추적 - 실패한 요청으로 인한 낭비 방지

알림 설정:

  • 일일 사용량이 기준치의 80% 초과 시 알림
  • 월별 예산의 90% 도달 시 경고
  • 비정상적인 사용 패턴 감지 시 즉시 알림

⚡ 성능 최적화

최적화 전략:

  • 캐싱 활용 - 동일한 요청 결과 재사용
  • 배치 처리 - 여러 요청을 한 번에 처리
  • 적절한 모델 선택 - 비용 대비 성능이 좋은 모델
  • 프롬프트 최적화 - 토큰 사용량을 줄이는 효율적인 프롬프트
💡 팁: 모니터링을 통해 워크플로우 성능을 지속적으로 개선하여 비용 효율성과 성능을 동시에 확보하세요.

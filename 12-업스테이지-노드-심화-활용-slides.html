🔬 업스테이지 노드 심화 활용

Document AI의 진짜 힘을 경험해보는 고급 활용법

  1. Document Parse 노드 마스터 - PDF/이미지의 복잡한 구조 완벽 해부
  2. Information Extract 노드 활용 - 비정형 문서에서 핵심 정보 자동 추출
  3. Classify 노드로 지능형 분류 - AI가 문서를 스스로 판단하고 분류
  4. 실전 활용 시나리오 - 계약서, 영수증, 이력서 처리 자동화

📄 Document Parse 노드 마스터

복잡한 PDF와 이미지 문서를 구조적으로 분석하는 고급 기능들을 살펴봅니다
🎯 고급 파싱 기능
  • 텍스트 추출: OCR로 이미지 속 텍스트 정확히 읽기
  • 표 구조 인식: 복잡한 테이블 데이터를 JSON으로 변환
  • 레이아웃 분석: 문서의 제목, 본문, 각주 구분
  • 좌표 정보: 각 텍스트의 정확한 위치 파악
⚙️ 파싱 옵션 설정
옵션 설명 활용 예시
OCR 이미지 텍스트 인식 스캔된 계약서
Table 표 데이터 추출 재무제표, 명세서
Layout 문서 구조 분석 보고서, 제안서
💡 실무 팁: PDF와 이미지가 섞인 문서는 Parse 노드 하나로 모든 정보를 추출할 수 있어 작업 효율이 획기적으로 향상됩니다.

🔧 실전 파싱 워크플로우

Document Parse 노드를 활용한 단계별 문서 처리 과정을 실습해보겠습니다
📋 파싱 워크플로우 구성
  1. 파일 입력
    • Read Binary File 노드로 PDF/이미지 파일 로드
    • HTTP Request로 URL에서 문서 다운로드
    • Gmail 첨부파일에서 자동 추출
  2. Document Parse 설정
    • 파싱 옵션 선택 (OCR, Table, Layout)
    • 언어 설정 (한국어 최적화)
    • 품질 설정 (정확도 vs 속도)
  3. 결과 데이터 처리
    • 텍스트 정제 및 후처리
    • 표 데이터를 구조화된 JSON으로 변환
    • 다음 단계 노드로 데이터 전달
// Parse 결과 데이터 구조 예시 { "text": "계약서 전체 텍스트 내용...", "tables": [ { "rows": [ ["항목", "금액", "비고"], ["컨설팅", "1,000,000원", "부가세 별도"] ] } ], "layout": { "title": "용역 계약서", "sections": ["제1조", "제2조", "제3조"] } }

🎯 Information Extract 노드 활용

비정형 문서에서 핵심 정보만 골라내는 지능형 추출 기능을 마스터합니다
📄 계약서 정보 추출
  • 당사자: 계약 주체 자동 식별
  • 계약 금액: 숫자와 통화 단위 파악
  • 계약 기간: 시작일과 종료일 추출
  • 특별 조건: 중요 약관 내용
🧾 영수증 정보 추출
  • 업체명: 상호와 사업자번호
  • 총 금액: 공급가액과 세액 분리
  • 거래 일시: 정확한 결제 시점
  • 결제 방법: 카드/현금 구분
📋 이력서 정보 추출
  • 개인 정보: 이름, 연락처, 주소
  • 경력 사항: 회사명, 근무 기간
  • 학력 정보: 학교명, 전공, 졸업
  • 기술 스택: 프로그래밍 언어, 도구
🔍 고급 팁: Information Extract는 프롬프트 엔지니어링을 활용해 원하는 정보 형태를 정확히 지정할 수 있어 맞춤형 추출이 가능합니다.

🤖 Classify 노드로 지능형 분류

AI가 문서의 내용과 특징을 분석해 자동으로 카테고리를 판단하는 시스템입니다
📂 문서 유형 자동 분류
문서 유형 분류 기준 후속 처리
계약서 조항, 서명, 날인 법무팀 검토
청구서 금액, 세금계산서 회계 시스템 입력
이력서 경력, 학력, 자기소개 인사팀 전달
보고서 데이터, 차트, 결론 관련 부서 배포
😊 감정 분석 및 우선순위 판단
  • 긴급도 분석: "긴급", "ASAP" 키워드 탐지
  • 감정 상태: 고객 불만, 만족, 중립 구분
  • 중요도 평가: VIP 고객, 대형 계약 식별
  • 자동 라우팅: 분류 결과에 따른 담당자 배정
⚡ Classify 워크플로우 설계
  1. 분류 기준 설정
    • 카테고리 정의와 예시 데이터 준비
    • 분류 정확도를 위한 키워드 라이브러리
  2. Switch 노드 연계
    • 분류 결과에 따른 자동 라우팅
    • 각 유형별 맞춤 처리 워크플로우 연결

🎯 통합 실전 시나리오

3개 업스테이지 노드를 연계한 완전 자동화 문서 처리 시스템을 구축해보겠습니다
🏢 스마트 계약서 처리 시스템
  1. 문서 입수 (Gmail Trigger)
    • 이메일 첨부파일 자동 감지
    • PDF/이미지 파일 필터링
    • 발송자 정보와 메일 내용 함께 저장
  2. 1단계: Parse (Document Parse)
    • PDF 전체 텍스트와 표 데이터 추출
    • 문서 레이아웃 분석으로 섹션 구분
  3. 2단계: Classify (Classify)
    • 계약서 유형 분류 (용역/매매/임대 등)
    • 긴급도와 중요도 자동 평가
  4. 3단계: Extract (Information Extract)
    • 계약 당사자, 금액, 기간 정보 추출
    • 특별 조항과 리스크 요소 식별
  5. 후속 처리 (자동화)
    • CRM 시스템에 고객 정보 업데이트
    • 계약 금액별 승인 라인 자동 설정
    • 법무팀/회계팀에 자동 알림 발송
    • 계약 만료 알림 스케줄 자동 등록
📊 성과 지표: 이 시스템으로 계약서 처리 시간을 평균 2시간에서 5분으로 단축하고, 인적 오류를 95% 감소시킬 수 있습니다.

🚪 Webhook - 나만의 똑똑한 대문

우리 집에 들어올 수 있는 특별한 대문을 만들어서 허가받은 사람만 "띵동!" 할 수 있게 하기
🏠 우리 집 대문 만들기
Webhook = 내 서비스로 들어오는 전용 대문
일상의 대문 Webhook 대문
우리 집 주소: 서울시 강남구... 내 서비스 주소: https://n8n.com/webhook/abc123
문패에 "김철수" 적혀있음 URL에 내 서비스 이름 포함
택배기사가 벨 누르면 즉시 알림 다른 서비스가 신호 보내면 즉시 반응
24시간 언제든 방문 가능 24시간 언제든 신호 수신 가능
🔐 허가받은 사람만 들어오게 하기
보안 = API Key / Auth 인증
집 보안 Webhook 보안
현관문 비밀번호 API Key 입력
출입증 확인 Auth 토큰 검증
아는 사람만 우리 집 주소 알려줌 신뢰하는 서비스만 Webhook URL 공유
수상한 사람은 차단 잘못된 인증은 자동 거부

🛠️ Webhook 실제 활용 사례

웹사이트 문의부터 주문 처리까지 즉시 자동화
🏃‍♂️ 고객 문의 즉시 대응 시스템
고객이 홈페이지에서 문의 작성 ↓ (즉시 Webhook 신호!) "새 문의 도착!" n8n이 바로 인식 ↓ 문의 내용을 업스테이지 AI가 분석 ("배송 문의네요!") ↓ 배송팀 이영희 과장에게 즉시 Slack 알림 ↓ 고객에게 "확인 중입니다" 자동 답장
🛒 온라인 주문
결제 시스템 → Webhook → 재고확인 → 포장팀 알림
👋 새 회원가입
가입 완료 → Webhook → 환영 이메일 → 쿠폰 발송
⭐ 제품 리뷰
리뷰 작성 → Webhook → 감정분석 → SNS 공유

⚙️ Webhook 노드 속성 설정 상세

전문가처럼 Webhook을 정교하게 설정해서 안전하고 효율적인 자동화 구축하기
🔗 Endpoint 설정
구분 용도 사용 시기
Test URL 개발 및 테스트용 워크플로우 제작 중
Production URL 실제 서비스용 배포 후 운영 시
🌐 HTTP Request 방식
  • GET: 데이터 가져오기 (조회)
  • POST: 데이터 업데이트 (생성/수정)
  • PUT: 전체 데이터 교체
  • DELETE: 데이터 삭제
🛡️ 보안 및 응답 설정
📍 Path 설정
  • 목적: 사용자 지정 URL 경로
  • 예시: /customer-inquiry, /order-update
  • 팁: 의미 있는 이름으로 관리 용이
🔐 Authentication (보안)
  • 권장: 보안을 위해 반드시 설정
  • API Key: 간단한 인증 방식
  • OAuth: 고급 인증 시스템
  • Basic Auth: 사용자명/비밀번호
📤 Respond 설정
  • 권장: 'Respond to Webhook' 노드 활용
  • 목적: 요청자에게 응답 메시지 전송
  • 예시: "접수 완료", "처리 중" 등
🎯 Webhook 설정 핵심: Test URL로 개발하고 → Authentication 설정으로 보안 강화 → Production URL로 실서비스! 'Respond to Webhook' 노드로 친절한 응답까지 완벽 구성하세요.

👆 메뉴얼 트리거 - 워크플로우 테스트 버튼

의도적으로 워크플로우 동작을 촉발시켜 당장 결과를 확인할 때 사용하는 테스트 도구
🧪 테스트 목적의 Manual Trigger
일상 테스트 워크플로우 테스트
새 전자제품 버튼 눌러보기 새 워크플로우 동작 확인
요리 중간에 맛 보기 중간 단계 결과 확인
드라이브 코스 미리 가보기 실제 서비스 전 테스트
새 앱 기능 체험해보기 데이터 흐름 점검
🚀 실제 테스트 시나리오
Schedule 대신 Manual 사용:
원래: 매일 오전 9시 자동 실행 ↓ 테스트: Manual Trigger로 지금 당장 실행 ↓ 결과: 기다리지 않고 즉시 동작 확인!
개발 과정에서 활용:
  • 워크플로우 로직 검증
  • 데이터 변환 결과 확인
  • API 연결 상태 점검
  • 오류 발생 지점 파악
💡 테스트 실행 방법
Schedule 워크플로우도 Manual로 테스트:
  • 원래는 정해진 시간에만 실행
  • Manual Trigger로 즉시 테스트 가능
  • 결과를 바로 확인해서 수정
  • 완성 후 Schedule로 변경
⚡ 시간을 기다리지 말고 바로 테스트해보세요!

💬 Chat Trigger - AI 대화 테스트 도구

Webhook 없이 바로 AI 시스템과 대화하며 테스트할 수 있는 간편한 도구
🖥️ Chat Trigger 화면
💬 화면 하단 채팅창을 통해서 대화하면서 테스트 가능함
⚡ Webhook 없이 바로 테스트하기
복잡한 설정 없이 즉시 시작
원래는 Webhook으로 외부에서 메시지를 받아야 하지만, Chat Trigger를 사용하면:
  • 즉시 테스트: 웹훅 설정 없이 바로 대화 가능
  • 채팅창 제공: 화면 하단에서 직접 입력
  • 실시간 확인: AI 응답과 워크플로우 진행 상황 즉시 확인
  • 반복 테스트: 다양한 문장으로 여러 번 테스트
💡 실제 활용 방법:
  • 고객 문의 시나리오 테스트
  • AI 응답 품질 검증
  • 워크플로우 각 단계 확인
  • 오류 발생 지점 실시간 파악

🔄 트리거에서 액션으로 - 신호에서 실행으로

트리거가 "시작!" 신호를 보내면 이제 실제 일을 처리하는 액션 노드들의 차례입니다
🚀 지금까지 배운 트리거들
  • Manual: 수동 실행 버튼
  • Schedule: 정해진 시간 자동 실행
  • Webhook: 즉시 반응 시스템
  • 앱 연동: 서비스 이벤트 감지
⚙️ 이제 배울 액션 노드들
  • 데이터 수집: HTTP, 데이터베이스에서 정보 가져오기
  • 데이터 가공: Set, Function으로 정보 변형
  • 데이터 저장: 처리된 정보를 안전하게 보관
  • 결과 전달: 완성된 결과를 사람들에게
🎯 액션 노드의 핵심: 요리하듯 3단계로 데이터를 처리합니다 - ①재료 준비 ②손질 가공 ③완성품 서빙

⚙️ 액션 노드들의 분류

자동화의 실제 작업을 담당하는 전문 직원들

🔧 작업 수행 노드
  1. 데이터 처리 - Set, Function
  2. 외부 서비스 연동 - CRM, 파일저장, SNS
🎛️ 흐름 제어 노드
  1. 조건부 실행 - IF, Switch, Filter
  2. 대기/중단 - Wait, Stop and Error
  3. 병합 및 나누기 - Merge, Split, Aggregate
🎯 액션 노드의 핵심: 작업 수행 노드는 실제 데이터를 처리하고 외부와 연동하며, 흐름 제어 노드는 워크플로우의 실행 순서와 조건을 관리합니다.

🔧 Set 노드 - 데이터 필드 조작의 스위스 칼

레고 블록을 조립하듯 데이터 필드를 자유자재로 추가, 수정, 삭제하는 만능 도구
🛠️ Set 노드의 주요 기능
기능 설명 예시
필드 추가 새로운 데이터 항목 생성 현재 시간, 계산값 추가
필드 수정 기존 데이터 값 변경 이름 형식 통일, 단위 변환
필드 삭제 불필요한 데이터 제거 개인정보, 임시 데이터 삭제
필드 이름 변경 데이터 키 이름 수정 한글명을 영문명으로
💡 Set 노드 활용 예시
🔹 Before (입력 데이터):
{ "name": "김철수", "age": 30, "raw_email": "KIM.CHEOLSU@COMPANY.COM" }
🔹 After (Set 노드 처리 후):
{ "name": "김철수", "age": 30, "email": "kim.cheolsu@company.com", "created_date": "2024-01-15", "status": "active", "age_group": "30대" }
  • 이메일을 소문자로 변환
  • 생성일자와 상태 추가
  • 나이를 바탕으로 연령대 계산
💡 Set 노드 핵심: 데이터를 '있는 그대로' 사용하지 말고, 필요에 맞게 가공해서 사용하세요! 단순 복사가 아닌 지능적 변환이 핵심입니다.

⚡ Function 노드 - 프로그래밍의 힘

Set 노드로 해결하기 어려운 복잡한 데이터 처리를 JavaScript/Python 코드로 자유자재로
🚀 Function 노드의 강력함
기능 Set 노드 Function 노드
필드 추가 ✅ 가능 ✅ 가능
조건부 처리 ❌ 제한적 ✅ if/else 자유자재
반복 처리 ❌ 불가능 ✅ for/while 반복
복잡한 계산 ❌ 제한적 ✅ 수학 함수 활용
API 호출 ❌ 불가능 ✅ fetch() 사용
외부 라이브러리 ❌ 불가능 ✅ npm 패키지
💡 Function 노드 활용 예시
🔹 복잡한 데이터 변환:
// 고객 등급 자동 계산 const customerData = items[0].json; const purchaseHistory = customerData.purchases; let grade = 'Bronze'; let totalAmount = 0; // 구매 금액 합계 계산 for(const purchase of purchaseHistory) { totalAmount += purchase.amount; } // 등급 결정 로직 if(totalAmount >= 1000000) { grade = 'Diamond'; } else if(totalAmount >= 500000) { grade = 'Gold'; } else if(totalAmount >= 100000) { grade = 'Silver'; } return { ...customerData, grade: grade, totalPurchase: totalAmount, nextGradeAmount: getNextGradeAmount(totalAmount) };
  • 구매 이력을 반복문으로 처리
  • 조건문으로 등급 자동 계산
  • 복잡한 비즈니스 로직 구현
💡 Function 노드 사용 팁: Set 노드로 80% 해결하고, 정말 복잡한 로직만 Function 노드로! 코드는 간단하게, 주석은 상세하게 작성하세요.

🎛️ 흐름 제어 노드 - 조건부 실행

사람이 판단하는 것처럼 상황에 따라 다른 길로 안내하는 교통 제어소
🚦 IF 노드 - 스마트한 교차로
조건 True (참) 동작 False (거짓) 동작
구매액 ≥ 10만원 VIP 고객으로 분류 일반 고객으로 분류
시간이 오전 9시 전 긴급 알림 발송 일반 알림 발송
파일 크기 > 5MB 압축 후 저장 그대로 저장
🌟 활용 예시:
고객 정보 입력 → IF 노드: "나이가 30세 이상인가?" → True: 성인 대상 마케팅 메일 → False: 청년층 대상 이벤트 정보
🔀 Switch & Filter 노드
🚌 Switch 노드 - 터미널 안내센터
  • 목적: 여러 조건 중 하나 선택
  • 예시: 지역별 담당자 배정
  • 서울 → 김과장, 부산 → 이대리
🔍 Filter 노드 - 엄격한 보안 검색대
  • 목적: 조건에 맞는 데이터만 통과
  • 예시: 미납 고객만 걸러내기
  • 조건: 결제상태 = "미납"
🔄 Loop 노드 - 반복 작업 자동화
  • 목적: 같은 작업 여러 번 반복
  • 예시: 고객 목록 전체에 메일 발송
💡 조건부 실행의 핵심: "만약~라면" 하는 인간의 사고를 자동화에 적용! 복잡한 비즈니스 로직도 단순한 조건 조합으로 해결 가능합니다.

⏸️ 흐름 제어 노드 - 대기/중단 & 병합/나누기

타이밍과 데이터 흐름을 완벽하게 제어하는 마에스트로의 지휘봉
⏰ 대기/중단 제어 노드
⏸️ Wait 노드 - 똑똑한 대기실
대기 타입 활용 예시
시간 대기 이메일 발송 후 5분 대기
조건 대기 결제 완료까지 대기
외부 신호 대기 승인 완료 신호 대기
🛑 Stop and Error 노드
  • 정상 중단: 작업 완료 후 안전하게 종료
  • 에러 중단: 문제 발생 시 즉시 정지
  • 알림 전송: 관리자에게 중단 이유 보고
📧 고객 문의 메일 도착 → 내용 분석 중 욕설 감지 → Stop and Error: "부적절한 내용 감지" → 관리자에게 즉시 알림 발송
🔀 병합/나누기 제어 노드
✂️ Split 노드 - 데이터 분배기
  • 목적: 하나의 데이터를 여러 경로로
  • 예시: 고객 목록을 지역별로 분할
  • 활용: 병렬 처리로 속도 향상
🔗 Merge 노드 - 데이터 합류점
  • 목적: 여러 경로의 데이터를 하나로
  • 예시: A팀, B팀 결과를 최종 보고서로
  • 활용: 분산 작업 결과 통합
📊 Aggregate 노드 - 데이터 요약 전문가
  • 합계: 월별 매출 총합
  • 평균: 고객 만족도 평균
  • 최대/최소: 최고/최저 판매량
  • 개수: 신규 고객 수
🎯 흐름 제어의 마스터키: Wait으로 타이밍 제어, Split/Merge로 병렬 처리, Aggregate로 데이터 요약! 복잡한 비즈니스 프로세스도 이것만 있으면 완벽 제어 가능합니다.

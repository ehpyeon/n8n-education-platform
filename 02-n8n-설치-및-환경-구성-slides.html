🏠 자동화 작업실 차리기 - 설치 방법 선택

새 사무실을 차릴 때 임대할지 구입할지 고민하듯, n8n도 설치 방법이 중요합니다

💻 로컬 설치 옵션

  • NPX - 가장 간단한 설치 방법
  • Docker - 컨테이너 기반 환경 독립성
  • 직접 서버 - 개인 서버나 VPS 설치

☁️ 클라우드 설치 옵션

  • n8n Cloud - 공식 클라우드 서비스 (유료)
  • AWS/GCP/Azure - 클라우드 플랫폼 배포
  • 서드파티 호스팅 - 관리형 서비스

🏆 내 주방 vs 배달주방 - 자체 호스팅 선택

집에서 직접 요리하듯, 내 컴퓨터에서 n8n을 운영하면 완전한 자유도를 얻습니다
비교 요소 ☁️ 클라우드 호스팅 (배달주방) 💻 자체 호스팅 (내 주방)
💰 비용 월 사용료 지속 지불 완전 무료 (전기료만)
🔒 데이터 보안 타사 서버에 데이터 저장 내 컴퓨터에 완전 보관
🎨 커스터마이징 제한된 설정 옵션 원하는 대로 모든 설정
📚 학습 효과 버튼만 클릭 서버 운영 경험 습득
👨‍🍳 결론: 내 주방에서 직접 요리하면 재료부터 조리법까지 모든 것을 자유롭게 조절할 수 있습니다!

⚡ NPX를 이용한 간단 설치 (추천)

가장 빠르고 간단한 방법으로 n8n을 시작할 수 있습니다
  1. Node.js 설치 확인
    node --version npm --version
  2. n8n 실행
    npx n8n
  3. 브라우저에서 접속
    • 주소: http://localhost:5678
    • 초기 계정 설정 진행
🎉 완료! 몇 분만에 n8n이 실행되어 브라우저에서 확인할 수 있습니다.

🐳 Docker를 이용한 설치 (고급 사용자)

컨테이너 환경으로 더 안정적이고 격리된 설치를 원할 때 사용합니다
  1. Docker 설치 확인
    docker --version
  2. n8n Docker 이미지 실행 (임시)
    docker run -it --rm \ --name n8n \ -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n \ n8nio/n8n
  3. 데이터 영구 저장 설정 (권장)
    docker run -d \ --restart unless-stopped \ --name n8n \ -p 5678:5678 \ -v n8n_data:/home/node/.n8n \ n8nio/n8n

🔧 기본 환경 설정

워크스페이스 구조를 이해하고 필요한 환경 변수를 설정합니다

📁 폴더 구조 이해

.n8n/ ├── config/ # 설정 파일 ├── nodes/ # 커스텀 노드 ├── workflows/ # 워크플로우 백업 └── credentials/ # 인증 정보 (암호화됨)

🌐 환경 변수 설정

export N8N_BASIC_AUTH_ACTIVE=true export N8N_BASIC_AUTH_USER=admin export N8N_BASIC_AUTH_PASSWORD=your_password export N8N_HOST=0.0.0.0 export N8N_PORT=5678

📂 필수 디렉토리 생성

mkdir -p ~/.n8n/workflows mkdir -p ~/.n8n/backups mkdir -p ~/.n8n/logs

🔧 자주 발생하는 설치 문제 해결

설치 과정에서 만날 수 있는 일반적인 문제들을 미리 대비합니다

🚫 포트 충돌 해결

# 포트 사용 확인 netstat -tulpn | grep 5678 # 다른 포트로 실행 npx n8n --port=8080

🔐 권한 문제 해결

# Node.js 전역 설치 권한 설정 sudo chown -R $(whoami) ~/.npm

💾 메모리 부족 문제

# Node.js 메모리 한도 증가 export NODE_OPTIONS="--max-old-space-size=4096"

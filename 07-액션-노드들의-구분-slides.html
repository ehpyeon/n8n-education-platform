🛒 1단계: 데이터 가져오기 (재료 준비)

좋은 요리를 위해서는 좋은 재료가 필요하듯, 좋은 자동화를 위해서도 좋은 데이터 수집이 필요
🌐 HTTP Request - 외부 배송 서비스
택시를 불러서 원하는 곳에서 재료를 가져오는 것과 같아요
실제 사용 예시:
가져올 정보 활용 방법
날씨 정보 사무실 스크린에 표시
환율 정보 실시간 환율 업데이트
주가 정보 투자 포트폴리오 모니터링
소셜미디어 인스타그램 포스트 정보
🗄️ 데이터베이스 - 회사 서류함
회사 서류함에서 필요한 문서를 찾는 것과 같아요
서류함 업무 데이터베이스 업무
고객 카드 찾기 고객 정보 조회 (Select)
새 고객 카드 추가 새 고객 등록 (Insert)
고객 정보 수정 기존 정보 변경 (Update)
퇴사자 카드 삭제 불필요 데이터 삭제 (Delete)

☁️ 클라우드 서비스 - 온라인 백화점

온라인 백화점에서 물건을 주문하듯 클라우드에 저장된 정보 가져오기
📊 자주 사용하는 클라우드 서비스
일상 예시 비즈니스 활용 노드
가계부 엑셀 파일 매출 데이터 가져오기 Google Sheets
배경화면 사진들 회사 자료 가져오기 Google Drive
에버노트 메모 업무 내용 가져오기 Notion
클라우드 동기화 폴더 파일 리스트 가져오기 Dropbox
🔑 주요 데이터베이스 노드
  • MySQL/PostgreSQL: 전통적인 데이터베이스
  • Supabase: 요즘 인기 클라우드 DB
  • Airtable: 사용하기 쉬운 클라우드 DB
💡 선택 기준:
  • 데이터 양과 복잡성
  • 팀의 기술 수준
  • 예산과 유지보수
  • 보안 요구사항

🔪 2단계: 데이터 변형하기 (재료 손질)

요리하기 전 재료를 다듬듯 데이터도 사용하기 좋게 가공이 필요
🥕 데이터 손질이 필요한 이유
요리 상황 데이터 상황
양파가 너무 커서 잘라야 함 고객 이름이 너무 길어서 줄여야 함
대파를 송송 썰기 가격을 원에서 달러로 변환
여러 재료를 섞어서 한 접시에 이름, 전화번호를 합쳐서 한 줄로
상한 재료 골라내기 좋은 고객만 목록에서 걸러내기
🔧 Set 노드 - 만능 나이프
주방의 만능 나이프처럼 대부분의 데이터 손질을 해결
Set 노드로 할 수 있는 작업:
  • 이름 조합: "김철수" + " 고객님"
  • 단위 변환: 1000원 × 1200 = 달러
  • 날짜 정리: "2024-03-15" → "3월 15일"
  • 중요 정보만 골라내기
장점: 코딩 없이 마우스 클릭만으로 설정!

👨‍🍳 Function 노드 - 전문 요리사 호출

Set 노드로 해결 안 되는 복잡한 작업은 전문 요리사에게 맡기기
🍜 전문 요리사가 필요한 상황
집에서는 못 만드는 요리를 전문 요리사에게 부탁하는 것처럼
Function 노드가 필요한 경우:
  • 복잡한 점수 계산 (할인율, 세금, 수수료)
  • 데이터 분류 (100명을 나이, 성별, 지역별로)
  • 고급 문자 처리 (이메일에서 중요 부분 추출)
  • 복잡한 조건 로직
🪄 초보자도 사용하는 마법
  1. 해야 할 작업을 ChatGPT에게 설명
  2. ChatGPT가 만든 코드 복사해서 붙여넣기
  3. 테스트해보고 안 되면 다시 요청
예시 요청:
"고객 나이를 계산해서 20대, 30대, 40대로 분류하는 JavaScript 코드를 만들어주세요"

🍽️ 3단계: 데이터 저장하기 (완성품 서빙)

완성된 요리를 상에 내놓듯 처리된 데이터를 적절한 곳에 저장
🗄️ 데이터베이스 저장
큰 창고에 보관
  • Supabase: 클라우드 DB
  • MySQL: 전통적 안정성
  • PostgreSQL: 고급 기능
용도: 고객정보, 주문내역, 재고데이터
☁️ 클라우드 서비스
모두가 볼 수 있는 공간
  • Google Sheets: 함께 수정하는 표
  • Google Drive: 문서, 사진 업로드
  • Notion: 팀 작업 공간
용도: 주간리포트, 업무대시보드
💬 커뮤니케이션 도구
사람들에게 따뜻하게 서빙
  • Slack: 팀 채터에서 즉시 알림
  • Gmail: 공식적인 이메일 발송
  • SMS: 긴급 알림
용도: 완료알림, 결과리포트

🎯 실제 업무 3단계 패턴

실제 비즈니스에서 자주 사용하는 데이터 처리 패턴들
📊 월말 리포트 자동 생성
상황: 여러 시스템의 데이터를 모아서 월말 리포트 작성
1단계: 5개 서비스에서 월간 데이터 수집 (재료 준비) ↓ 2단계: Function 노드로 통계 계산하고 차트 만들기 (재료 손질) ↓ 3단계: Google Sheets로 리포트 업로드 + 사장님께 이메일 (완성품 서빙)
🎂 생일 축하 이벤트 자동화
상황: 오늘이 생일인 고객들에게 개인화된 축하 메시지
1단계: 고객 DB에서 오늘 생일인 고객들 찾기 (재료 준비) ↓ 2단계: Set 노드로 각 고객명 + 축하 메시지 조합 (재료 손질) ↓ 3단계: Gmail로 각 고객에게 개인화된 이메일 발송 (완성품 서빙)
🍽️ 무료급식소 운영 (100명 처리)
상황: 하루에 100명의 신청자 데이터를 처리해서 담당자에게 배정
1단계: Google Sheets에서 신청자 명단 가져오기 (재료 준비) ↓ 2단계: Set 노드로 반별 배정하고 알림 메시지 작성 (재료 손질) ↓ 3단계: Slack으로 각 반 담당자에게 알림 발송 (완성품 서빙)
🎯 핵심 포인트: 이제 3단계 패턴을 이해하셨다면, 어떤 복잡한 업무도 논리적으로 자동화할 수 있습니다. 다음 강의에서는 더 심화된 데이터 저장 기술들을 알아보겠습니다.

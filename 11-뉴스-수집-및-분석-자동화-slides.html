🧠 2단계: AI 기반 요약 및 임베딩 처리

사서가 책을 요약하고 분류하듯 AI가 뉴스를 요약하고 검색 가능한 형태로 변환
🤖 Structured Output 기반 AI 요약 처리
LLM Structured Output 활용:
  • 정형화된 출력: JSON 스키마 정의로 일관된 형식
  • 자동 추출: 핵심 정보, 기업, 키워드
  • 영향도 평가: 1-3 점수로 중요도 수치화 (높을수록 중요)
  • 이벤트 분류: Launch, Update, Funding 등 12가지 유형
Structured Output 스키마:
{ "summary_ko": "400~900자 사실 요약", "event_type": "Launch|Update|Partnership|Funding|Policy|Research|Benchmark|Security|M&A|Hiring|Ecosystem|Other", "companies": ["회사명1", "회사명2", ...], "impact_score": 3 // 1~3 정수 (점수가 클수록 중요) }
실제 출력 예시:
{ "summary_ko": "삼성전자가 2024년 1분기 실적을 발표했다. 매출은 전년 대비 13.4% 증가한 71조 원을 기록했으며, 반도체 부문은 적자 폭이 전분기 대비 크게 축소되었다. 특히 AI 서버용 HBM 메모리 수요 증가와 메모리 가격 상승세가 실적 개선의 주요 요인으로 분석된다...", "event_type": "Update", "companies": ["삼성전자", "SK하이닉스"], "impact_score": 3 }
🧠 임베딩 벡터 변환
AI 요약된 summary_ko 필드를 벡터로 변환하여 의미 기반 검색 활성화
벡터화 대상 필드:
summary_ko 필드 (400~900자 요약문) ↓ 🔄 Upstage Embedding API 변환 ↓ [0.245, -0.123, 0.678, 0.891, ...] (1536차원 벡터)
벡터화 과정 예시:
입력: "삼성전자가 2024년 1분기 실적을 발표했다. 매출은 전년 대비 13.4% 증가한 71조 원을 기록했으며..." 💡 의미 이해: - "실적"과 "수익"이 유사 - "반도체"와 "칩"이 연관 - "회복"과 "개선"이 비슷한 의미
벡터화의 장점:
  • 의미 기반 검색: 단어가 달라도 의미가 같으면 검색
  • 유사 기사 연결: 관련 뉴스 자동 발견
  • 맥락 이해: 뉘앙스와 문맥 파악
  • 다국어 지원: 언어 간 의미 매칭 가능

💾 Qdrant에 저장되는 데이터 구조
Vector 데이터:

Metadata 필드 구조:
필드명 출처 설명
title RSS 뉴스 제목
link RSS 원본 기사 URL
author RSS 작성자
pubDate RSS 게시 날짜/시간
category RSS 뉴스 카테고리
eventType AI 분석 Launch, Update, Funding 등
companies AI 분석 언급된 회사/기관명 배열
impact_score AI 분석 영향도 점수 (1~3)
🎯 하이브리드 활용: Vector로 의미 검색 + Metadata로 정확한 필터링 (예: eventType="Funding", impact_score≥2)

🔍 4단계: 뉴스 분석 요청 처리 시스템

연구원이 자료실에서 관련 문헌을 찾듯 사용자 질문에 맞는 뉴스들을 검색하여 분석
🔍 하이브리드 검색 방식
1️⃣ 벡터 검색 (의미 기반)
특징 설명
검색 방식 summary_ko 임베딩 벡터 유사도 계산
장점 단어가 달라도 의미가 같으면 검색
활용 "실적 개선"과 "수익 증가"를 연결
2️⃣ 관련도 검색 (메타데이터 필터링)
특징 설명
검색 방식 메타데이터 필드로 정확한 조건 매칭
장점 특정 조건 완벽 필터링
활용 eventType, companies, impact_score 등
🎯 두 방식 동시 사용: 벡터로 의미 검색 + 메타데이터로 조건 필터링 = 최적의 검색 결과
🔑 메타데이터 필터링 검색 예시
검색 요구사항:
"신제품 출시(Launch) 뉴스만 찾기"
  • 이벤트 타입이 Launch (eventType = "Launch")
1. 키워드 필드 인덱싱 설정:
PUT /collections/{collection}/index { "field_name": "eventType", "field_schema": { "type": "keyword" } }
2. eventType 필터링 검색:
POST /collections/{collection}/points/query { "query": [0.245, -0.123, ...], // 질문 임베딩 벡터 "filter": { "must": [ { "key": "eventType", "match": { "value": "Launch" } } ] }, "limit": 10 }
💡 실전 활용: 벡터 검색으로 "신제품"과 의미적으로 유사한 뉴스를 찾고, eventType 필터로 Launch 이벤트만 정확하게 추출!

🤖 나만의 복합 Agent 봇 제작

RAG + 문서처리 + Agent를 통합한 실무형 자동화 시스템

  1. 시스템 기획 - 전체 아키텍처 설계 및 기능 정의
  2. 러버블 프론트엔드 - 간단한 웹 인터페이스로 외부 연결
  3. 도구 통합 실습 - RAG, 문서처리, 정보검색 도구들을 Agent에 연결

📋 1단계: 시스템 기획 및 설계

복합 Agent 봇의 전체 기능과 구조를 구체적으로 설계합니다
🎯 기능 요구사항 정의
기능 목적 연결 도구
문서 분석 PDF/이미지 텍스트 추출 및 정리 Upstage Doc Parse
지식 검색 회사 문서 기반 정보 제공 RAG Vector Store
데이터 파싱 구조화된 정보로 변환 JSON/표 형태 변환
정보 검색 실시간 외부 정보 조회 웹 검색 API
🏗️ 아키텍처 설계 원칙
  • 모듈화: 각 기능을 독립적 서브워크플로우로 분리
  • 확장성: 새로운 도구 추가가 쉬운 구조
  • 재사용성: 다른 프로젝트에도 활용 가능
  • 성능: 병렬 처리 및 캐싱 적용
  • 안정성: 에러 처리 및 복구 로직
💡 핵심 아이디어: Agent가 사용자 요청을 분석해서 필요한 도구들을 자동으로 선택하고 조합
📝 기획 체크리스트
  1. 사용 시나리오 정의
    • 주요 사용 케이스 3-5가지 도출
    • 사용자 흐름(User Flow) 설계
    • 입력/출력 형태 정의
  2. 기술 스택 선택
    • Agent: OpenAI GPT-4 기반 n8n Agent
    • 프론트엔드: 러버블(Lovable)
    • 백엔드: n8n 서브워크플로우
  3. 데이터 흐름 설계
    • 각 단계별 데이터 변환 과정
    • 에러 처리 및 예외 상황 대응
    • 성능 최적화 지점 파악

🌐 2단계: 러버블로 프론트엔드 구축

간단한 웹 인터페이스를 만들어 사용자와 Agent를 연결합니다
🎨 러버블 기본 설정
  1. 프로젝트 생성
    • 러버블 계정 생성 및 로그인
    • 새 프로젝트 "Agent Bot Interface" 생성
    • 기본 템플릿 선택
  2. UI 컴포넌트 구성
    • 채팅 인터페이스 생성
    • 파일 업로드 영역 추가
    • 응답 표시 영역 설정
  3. 스타일링 적용
    • 간단하고 직관적인 디자인
    • 모바일 반응형 적용
    • 로딩 상태 표시
🔗 n8n 웹훅 연결
// 러버블에서 n8n 웹훅 호출 const sendToAgent = async (message, file) => { const webhookUrl = "https://n8n.upstage.ai/webhook/agent-bot"; const response = await fetch(webhookUrl, { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ message: message, file_url: file ? await uploadFile(file) : null, timestamp: new Date().toISOString() }) }); return await response.json(); };
💡 실제 흐름: 사용자 입력 → 러버블 → 웹훅 → n8n Agent → 응답
🚀 러버블 구현 실습

📱 인터페이스 구성

  • 입력 영역: 텍스트 + 파일 업로드
  • 채팅 영역: 대화 히스토리 표시
  • 상태 표시: 처리 중/완료 알림
  • 결과 영역: 구조화된 응답 표시

⚙️ 기능 구현

  • 실시간 통신: WebSocket 또는 Polling
  • 파일 처리: 이미지/PDF 업로드 지원
  • 응답 파싱: JSON 데이터 시각화
  • 에러 처리: 네트워크 오류 대응

📄 3단계: 문서 처리 도구 통합

Upstage Doc Parse를 Agent 도구로 연결하여 문서 분석 기능을 구현합니다
🔧 문서 처리 서브워크플로우 생성
  1. 새 워크플로우 생성
    • 이름: "Document_Processor"
    • 시작: Execute Workflow Trigger 노드
    • 입력: file_url, output_format
  2. Upstage 노드 설정
    • Document Parse 노드 추가
    • API 키 및 엔드포인트 설정
    • 파일 형식 자동 감지
  3. 결과 후처리
    • 텍스트 정제 및 구조화
    • 출력 형식 변환 (JSON/표/요약)
    • 응답 데이터 표준화

🔍 RAG 지식 검색 도구 통합

기존 Vector Store를 Agent 도구로 연결하여 지능형 지식 검색 기능을 구현합니다
🗄️ RAG 검색 서브워크플로우 생성
  1. 새 워크플로우 생성
    • 이름: "Knowledge_Search"
    • 입력: query, top_k, filter
    • 기존 Vector Store 활용
  2. 검색 로직 구성
    • Pinecone/Qdrant Vector Store 연결
    • 임베딩 생성 및 유사도 검색
    • 검색 결과 랭킹 및 필터링
  3. 결과 후처리
    • 관련도 점수 기반 정렬
    • 출처 정보 추가
    • 컨텍스트 길이 최적화
🤖 Agent에 RAG 도구 등록
# Knowledge Search Tool 설정 { "name": "knowledge_search", "description": "회사 내부 문서와 지식베이스에서 관련 정보를 검색합니다. 정책, 가이드라인, FAQ 등에서 정확한 답변을 찾을 수 있습니다.", "parameters": { "query": { "type": "string", "description": "검색할 질문이나 키워드" }, "top_k": { "type": "number", "default": 3, "description": "가져올 결과 개수" }, "category": { "type": "string", "description": "문서 카테고리 필터 (선택사항)" } } }
💡 사용 예시: "회사 휴가 정책이 어떻게 되나요?"
🚀 실습: RAG 도구를 Agent에 연결하기

🔍 검색 품질 최적화

  • 임베딩 모델: 한국어 특화 모델 선택
  • 청크 크기: 500-1000자로 조정
  • 유사도 임계값: 0.7 이상 결과만 반환
  • 메타데이터: 문서 제목, 날짜, 카테고리 포함

🧪 통합 테스트

  • 검색 정확도: 관련 문서 찾기 테스트
  • 응답 품질: 출처와 함께 답변 확인
  • Agent 연동: 자동 도구 선택 검증
  • 복합 쿼리: 문서+검색 동시 요청

📊 데이터 파싱 및 변환 도구 통합

비구조화된 데이터를 구조화된 형태로 변환하는 파싱 도구를 Agent에 추가합니다
⚙️ 데이터 파싱 서브워크플로우
  1. 새 워크플로우 생성
    • 이름: "Data_Parser"

  2. 파싱 로직 구성
    • 정규표현식 패턴 매칭
    • JSON/CSV/XML 파싱
    • 구조화 스키마 적용
  3. 결과 변환
    • 테이블 형태로 정리
    • 차트 데이터 생성
    • 요약 통계 추가

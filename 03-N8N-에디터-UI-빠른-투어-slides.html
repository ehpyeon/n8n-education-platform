🖥️ N8N 에디터 UI 완전 가이드

회사 사무실처럼 체계적으로 구성된 5개 핵심 화면을 마스터하자

  1. 메인 워크플로우 관리 화면 - 사장실에서 전체 프로젝트 관리
  2. 크레덴셜 관리 화면 - 출입카드와 열쇠 보관함
  3. Execution 관리 화면 - 작업 진행 상황 모니터링실
  4. 세팅 화면 - 사무실 환경 설정 및 도구 추가
  5. 워크플로우 제작 화면 - 실제 작업을 수행하는 작업실

🏢 메인 워크플로우 관리 화면

회사의 사장실처럼 모든 프로젝트와 자동화를 한눈에 관리하는 중앙 통제실입니다

📊 Overview vs Personal 작업공간

구분 Overview Personal
역할 전체 팀 프로젝트 관리 개인 작업 공간
비유 회사 전체 관리실 내 개인 책상
주요 기능 모든 워크플로우 조회 내 워크플로우만 관리

🎛️ 워크플로우 관리 기능

  • 워크플로우 목록 - 파일 탐색기처럼 정리된 리스트
  • Active 상태 제어 - 자동화 ON/OFF 스위치
  • 공유하기 - 동료에게 워크플로우 전달
  • 복제하기 - 템플릿으로 새 워크플로우 생성
  • 보관하기 - 사용하지 않는 워크플로우 정리
  • 검색 필터 - 이름, 상태별 빠른 찾기

✨ 워크플로우 생성과 라이프사이클 관리

프로젝트 폴더를 만들고 관리하는 것처럼 체계적으로 워크플로우를 운영합니다

📁 새 워크플로우 생성

  1. 생성하기
    • Overview/Personal에서 Create Workflow 클릭
    • 새 폴더 만들기와 동일한 개념
  2. 명명규칙
    • "고객문의-자동응답-v1" 같은 의미있는 이름
    • Ctrl + S로 저장
  3. 복제 활용
    • 기존 워크플로우를 템플릿으로 복제
    • Duplicate → 이름 변경 → 수정

🔄 Active 상태 관리

  • Inactive (비활성) - 테스트 중이거나 개발 단계
  • Active (활성) - 실제 운영 중인 자동화
  • 상태 전환 - 우상단 토글 스위치로 간편 제어
  • 주의사항 - Active 상태에서만 트리거 동작

📤 공유 및 백업

  • Share - 팀원에게 읽기 전용 공유
  • Duplicate - 완전 복사본 생성
  • Archive - 사용하지 않는 워크플로우 보관
  • History - 버전 관리 및 복원

🔐 크레덴셜 관리 화면

회사의 열쇠보관함처럼 외부 서비스 접근을 위한 인증 정보를 안전하게 관리합니다

🗝️ 크레덴셜이란?

실생활 비유 크레덴셜 종류 용도
🏠 집 열쇠 API Key 개인 서비스 전체 접근
🎫 방문증 OAuth 토큰 제한된 권한으로 안전 접근

📂 크레덴셜 관리 인터페이스

  • Credentials 메뉴 - 좌측 사이드바에서 접근
  • My credentials - 내가 생성한 인증 정보
  • All credentials - 팀 공유 인증 정보 포함
  • Create credential - 새 인증 정보 추가
  • Test connection - 연결 상태 확인
  • 암호화 저장 - 민감 정보 자동 보호

🔑 API Key vs OAuth - 두 가지 인증 방식

집 열쇠와 방문증처럼 각각 다른 보안 수준과 편의성을 제공합니다

🗝️ API Key 방식 (집 열쇠)

  • 특징 - 한 번 발급받으면 영구 사용
  • 보안 - 전체 권한, 노출 시 위험
  • 설정 - 간단한 복사&붙여넣기
  • 관리 - 정기적 교체 권장

주요 사용 서비스:

  • OpenAI (ChatGPT API)
  • Upstage (한국어 AI)
  • 대부분의 AI 서비스
  • 날씨, 뉴스 등 공개 API

인증 절차:

sequenceDiagram participant 사용자 participant API제공업체 participant n8n 사용자->>API제공업체: API Key 요청 API제공업체->>사용자: API Key 발급 사용자->>n8n: API Key 입력 n8n->>API제공업체: API Key로 인증 및 호출

실제 설정 화면:

OpenAI API Key를 직접 입력하여 인증하는 방식입니다.

🎫 OAuth 방식 (방문증)

  • 특징 - 임시 토큰, 자동 갱신
  • 보안 - 제한된 권한, 언제든 철회 가능
  • 설정 - "Connect" 버튼으로 간편 연결
  • 관리 - 자동 토큰 관리

주요 사용 서비스:

  • Google (Gmail, Sheets, Drive)
  • Microsoft (Outlook, Excel)
  • 소셜미디어 (Twitter, LinkedIn)
  • 클라우드 스토리지 (Dropbox)

인증 절차:

sequenceDiagram participant 사용자 participant OAuth제공업체 participant n8n 사용자->>n8n: OAuth 연결 요청 n8n->>OAuth제공업체: 인증 페이지로 리다이렉트 사용자->>OAuth제공업체: 로그인 및 권한 승인 OAuth제공업체->>n8n: 토큰 발급 n8n->>OAuth제공업체: 토큰으로 서비스 호출

실제 설정 화면:

Google Drive에 OAuth 방식으로 연결하는 방식입니다.

🔧 Execution 상세 디버깅 (심화)

의사가 환자를 진단하듯이 개별 실행 문제를 단계별로 해결합니다

🖥️ 상세 디버깅 화면

실패한 워크플로우의 실행 상태와 각 노드별 에러 정보를 상세히 분석할 수 있습니다.

🔍 체계적인 디버깅 프로세스

  1. 실패한 실행 확인
    • Executions 탭에서 빨간색 실패 표시 확인
    • 실행 시간과 에러 개수 파악
  2. 실패한 노드 확인
    • 워크플로우에서 빨간색으로 표시된 노드 식별
    • 에러가 발생한 지점 파악
  3. 상세 원인 확인
    • 실패 노드 클릭으로 에러 메시지 확인
    • 입력/출력 데이터 검토
    • API 응답 코드 및 상세 에러 로그 분석
  4. 문제 해결 후 Replay
    • 워크플로우 수정 후 저장
    • Replay 버튼으로 동일 조건 재실행
    • 성공할 때까지 반복 디버깅
💡 디버깅 팁: Replay 기능을 통해 같은 입력 데이터로 반복 테스트하며 문제를 해결할 수 있습니다.

📊 Execution 관리 화면

공장의 생산 현황판처럼 시스템 전반의 건강 상태를 한눈에 모니터링합니다

📈 시스템 전반 통계 확인

  • 성공률 - 전체 시스템 안정성 지표
  • 실패 수 - 문제 발생 빈도 확인
  • 실패율 - 시스템 이상 여부 판단
  • 트렌드 분석 - 시간대별 패턴 파악

🔍 실행 목록 모니터링

  • 성공/실패 상태 - 초록색/빨간색 표시로 즉시 구분
  • 필터링 기능 - 실패 사례만 모아서 확인
  • 워크플로우별 현황 - 어느 자동화에서 문제 발생하는지 파악
  • 실행 시간 확인 - 언제 문제가 발생했는지 추적
🎯 모니터링 전략: 통계로 전반적인 시스템 상태를 파악하고, 실행 목록에서 구체적인 문제 지점을 식별합니다.

🔍 실행 모니터링과 트러블슈팅

의사가 환자를 진단하듯이 체계적으로 문제를 찾고 해결합니다

📈 실행 패턴 분석

  • 성공률 추이 - 시간대별 성공/실패 비율
  • 실행 빈도 - 트리거 발생 패턴 분석
  • 처리 시간 - 평균 실행 시간 모니터링
  • 병목 지점 - 가장 오래 걸리는 노드 식별

⚠️ 알림 설정

  • 실패율이 20% 초과 시 Slack 알림
  • 5분 이상 실행되지 않은 워크플로우 체크
  • 크리티컬 워크플로우 실패 시 즉시 알림

문제 상황 해결 방법

👥 핵심 설정 기능

사용자 관리

  • User Settings - 개인 프로필, 언어, 시간대 설정
  • Password Change - 보안을 위한 정기 변경
  • API Keys - n8n API 접근용 개인 키

Community Node 설치

앱스토어처럼 필요한 서비스 연동 노드를 추가

  1. Community Nodes 메뉴 선택
  2. Browse 버튼으로 노드 검색
  3. 필요한 노드 (Notion, Discord 등) Install

기타 설정

  • 보안 설정 - 2단계 인증, 세션 관리
  • 시스템 설정 - 로그 수준, 캐시 관리
💡 중요: Community Node 설치 후 n8n 재시작이 필요할 수 있습니다.

🔌 Community Node 설치 과정

스마트폰에 새 앱을 설치하듯이 필요한 서비스 연동 노드를 추가합니다
  1. 설정 메뉴 접근
    • 좌측 사이드바에서 Settings 클릭
    • Community Nodes 섹션 선택
  2. 커뮤니티 노드 이름 입력
    • 노드 패키지 이름 입력 (예: "n8n-nodes-solar")
    • 정확한 패키지명 확인
  3. 설치 실행
    • "Install" 버튼 클릭
    • 설치 진행 상황 모니터링
  4. 설치 완료 확인
    • 페이지 새로고침
    • 노드 패널에서 새 노드 확인
⚠️ 주의사항: Community Node는 제3자가 개발한 것으로 공식 지원이 제한될 수 있습니다. 신뢰할 수 있는 개발자의 노드를 선택하세요.

🎨 워크플로우 제작 화면

실제 자동화를 만드는 핵심 작업 공간입니다
💡 워크플로우 제작의 핵심: 캔버스에서 노드를 드래그하여 연결하고, 우측 패널에서 필요한 노드를 찾아 추가할 수 있습니다. 실제 작업이 이루어지는 가장 중요한 화면입니다.

🎨 워크플로우 제작 화면 개요

실제 자동화를 만드는 작업실로 모든 창작 활동이 이루어지는 핵심 공간입니다

🖼️ 캔버스 작업공간

  • 점선 격자 배경 - 노드 배치를 위한 가이드
  • 무한 스크롤 - 복잡한 워크플로우도 자유롭게 확장
  • 줌 인/아웃 - 세부 작업과 전체 조망 자유 전환
  • 드래그 앤 드롭 - 직관적인 노드 배치
  • 자동 정렬 - 깔끔한 레이아웃 자동 생성

🎮 캔버스 조작법

  • Ctrl + 클릭 드래그 - 캔버스 이동
  • 마우스 휠 - 줌 인/아웃
  • 스페이스바 - 캔버스 전체 보기

🧩 우측 노드 패널

  • 열기 방법 - + 버튼 또는 Tab 키
  • 카테고리별 분류 - 용도에 따른 체계적 구성
  • 검색 기능 - 원하는 노드 빠른 찾기
  • 드래그 설치 - 패널에서 캔버스로 끌어오기

주요 노드 카테고리:

  • Triggers - 자동화 시작점
  • Actions - 실제 작업 수행
  • Advanced AI - AI 기능 활용
  • Data transformation - 데이터 가공
  • Flow control - 흐름 제어

📝 메모 기능과 개발 중 실행 모니터링

복잡한 워크플로우도 체계적으로 관리하고 실시간으로 테스트할 수 있습니다

📋 메모/스티키 노트 활용

  • 추가 방법 - 캔버스에서 노트 아이콘 클릭
  • 용도 - 설계 의도, 주의사항, TODO 기록
  • 색상 구분 - 우선순위나 카테고리별 분류
  • 크기 조절 - 내용량에 맞는 자유로운 크기

메모 활용 예시:

  • "고객 데이터 처리 구간 - 개인정보 주의"
  • "API 제한: 분당 100회까지"
  • "TODO: 에러 처리 로직 추가 필요"
  • "v2 업데이트 시 이 부분 개선"

⚡ 개발 중 실행 모니터링

  • 실시간 실행 - Execute Workflow로 전체 테스트
  • 개별 노드 실행 - 특정 노드만 단독 테스트
  • 데이터 흐름 확인 - 각 연결선의 데이터 전달 상태
  • 에러 표시 - 빨간색으로 문제 노드 즉시 식별

디버깅 팁:

  • Set 노드로 중간 데이터 확인
  • Console 노드로 로그 출력
  • If 노드로 조건별 분기 테스트
  • Try/Catch로 에러 처리 확인

💾 Save 버튼과 Active 토글 관리

자동차의 시동과 주차브레이크처럼 반드시 확인해야 할 핵심 컨트롤입니다

💾 Save 버튼의 중요성

  • 자동 저장 없음 - 반드시 수동으로 저장 필요
  • 저장 타이밍 - 노드 추가/수정 후 즉시 저장
  • 단축키 - Ctrl + S (윈도우), Cmd + S (맥)
  • 저장 표시 - 제목 옆 별표(*)로 미저장 상태 확인

저장하지 않으면:

  • 브라우저 새로고침 시 모든 작업 손실
  • 다른 워크플로우로 이동 시 변경사항 사라짐
  • 실행 시 이전 버전으로 동작
⚠️ 습관화 필수: 노드 하나 추가할 때마다 Ctrl+S 습관을 만드세요!

  2. 에러 처리 로직 확인

🚀 5개 핵심 화면을 활용한 완벽한 워크플로우 개발 프로세스

회사 업무 프로세스처럼 체계적으로 5개 화면을 순환하며 자동화를 완성합니다
  1. 📊 메인 관리 화면: 프로젝트 시작
    • Overview/Personal에서 "Create Workflow" 클릭
    • 의미있는 이름으로 워크플로우 생성
    • 템플릿이 있다면 Duplicate 활용
  2. 🔐 크레덴셜 화면: 인증 설정
    • 필요한 외부 서비스 크레덴셜 미리 준비
    • API Key 또는 OAuth 방식으로 연결
    • Test connection으로 연결 상태 확인
  3. 🎨 워크플로우 제작 화면: 실제 구축
    • 트리거 노드부터 시작하여 순차적 구성
    • 각 노드마다 Ctrl + S로 즉시 저장
    • 메모 노드로 설계 의도 기록
    • 개별 노드 실행으로 단계별 테스트
  4. 📊 Execution 화면: 테스트 및 디버깅
    • Execute Workflow로 전체 테스트 실행
    • 실패 시 에러 로그 확인 및 수정
    • Replay 기능으로 반복 테스트
  5. 📊 메인 관리 화면: 배포 및 운영
    • 최종 저장 후 Active 토글 ON
    • Execution 화면에서 실행 모니터링
    • 필요시 ⚙️ 세팅 화면에서 Community Node 추가
🎯 UI 마스터 완료! 이제 5개 핵심 화면을 자유자재로 활용하여 어떤 자동화든 체계적으로 구축할 수 있습니다. 다음 강의에서는 실제 노드 연결과 데이터 흐름을 배워보겠습니다!

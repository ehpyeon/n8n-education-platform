🏗️ 워크플로우의 핵심 구성 요소

📌 노드 (Node)

  • 트리거: 워크플로우를 시작하는 신호
  • 액션: 실제 작업을 수행하는 처리 블록

🔗 연결 (Connection)

  • 노드 간 데이터 전달 통로
  • 순서 보장 및 실행 흐름 제어
  • JSON 형태로 정보 전송
🎯 핵심 포인트: 워크플로우는 개별 작업들을 트리거-액션-연결선으로 엮어 전체 업무 프로세스를 자동화하는 시스템입니다.

🧩 노드란? - 워크플로우의 핵심 블록

노드는 특정 작업을 수행하는 독립적인 단위입니다
🏗️ 노드의 핵심 특징
  • 독립성: 고유한 기능 담당
  • 재사용성: 여러 워크플로우에서 활용
  • 설정 가능성: 세부 옵션 커스터마이징
  • 확장성: 새로운 노드 추가 가능
🔄 노드 동작 원리
입력 데이터 → [노드 처리] → 출력 데이터 ↓ JSON 형태로 다음 노드에 전달 ↓ 워크플로우 체인 형성

🚀 트리거 노드 - 워크플로우의 시작점

워크플로우를 시작하는 다양한 신호와 방법들
⚡ 이벤트 기반
  • Webhook: HTTP 요청 수신
  • Email Trigger: 새 이메일 도착
  • File Trigger: 파일 변경 감지
  • Database Trigger: DB 변경 반응
⏰ 시간 기반
  • Cron: 정밀한 스케줄링
  • Interval: 간격 기반 실행
  • Schedule: 특정 시간 실행
👆 수동 트리거
  • Manual Trigger: 직접 실행
  • Chat Trigger: 대화 시작점

🎛️ 액션 노드 - 흐름 제어

워크플로우의 실행 흐름을 제어하고 조건부 처리를 담당
🔀 조건부 실행
  • IF: 참/거짓 조건 분기
  • Switch: 다중 조건 분기
  • Filter: 조건에 맞는 데이터 필터링
⏸️ 대기/종료
  • Wait: 지연 시간 설정
  • Stop and Error: 워크플로우 중단
  • No Operation: 빈 처리 단계
🔄 병합 및 나누기
  • Merge: 여러 데이터 스트림 병합
  • Split In Batches: 대용량 데이터 배치 처리
  • Item Lists: 배열 데이터 분할/결합

🔗 연결 유형 - 데이터 흐름의 경로

노드 간 데이터 전달과 에러 처리를 위한 다양한 연결 방식
✅ Main Connection
정상적인 데이터 처리 경로
  • 성공적인 노드 실행 후 전달
  • 주요 비즈니스 로직 경로
  • 데이터 변환과 처리의 통로
❌ Error Connection
오류 발생 시 처리 경로
  • 예외 상황 대안 처리
  • 에러 로깅 및 알림
  • 복구 메커니즘 구현
🔀 조건부 Connection
조건에 따른 분기 처리
  • IF 노드의 True/False
  • Switch 노드의 다중 분기
  • API 응답 상태별 처리

🌐 분기 패턴 - 워크플로우 설계 전략

비즈니스 요구사항에 맞는 효율적인 워크플로우 구조
📝 Sequential Flow (순차적)
graph LR A[🚀 Trigger] --> B[⚙️ Process A] B --> C[🔧 Process B] C --> D[💾 Process C] D --> E[✅ End] style A fill:#e1f5fe style B fill:#f3e5f5 style C fill:#f3e5f5 style D fill:#f3e5f5 style E fill:#e8f5e8
활용 사례:
  • 데이터 검증 → 변환 → 저장
  • 사용자 입력 → 분석 → 응답
  • 파일 수집 → 처리 → 배포
⚡ Parallel Flow (병렬)
graph LR A[🚀 Trigger] --> B[🔀 Split] B --> C[⚙️ Process A] B --> D[🔧 Process B] B --> E[💾 Process C] C --> F[🔗 Merge] D --> F E --> F F --> G[✅ End] style A fill:#e1f5fe style B fill:#fff3e0 style C fill:#f3e5f5 style D fill:#f3e5f5 style E fill:#f3e5f5 style F fill:#fff3e0 style G fill:#e8f5e8
활용 사례:
  • 다중 API 동시 호출
  • 여러 채널 알림 발송
  • 독립적 데이터 처리 병렬화
🔄 Conditional Flow (조건부)
graph LR A[🚀 Trigger] --> B{🔍 IF 조건} B -->|True| C[✅ True Path] B -->|False| D[❌ False Path] C --> E[✅ End] D --> E style A fill:#e1f5fe style B fill:#fff3e0 style C fill:#e8f5e8 style D fill:#ffebee style E fill:#f8f9fa
활용 사례:
  • 사용자 권한별 다른 처리
  • 데이터 값 기반 분기 로직
  • 시간/날짜 조건부 작업
🔁 Loop Flow (반복)
graph LR A[🚀 Start] --> C{🔍 Decision} C -->|Continue| B[⚙️ Action 1] B --> F[🔧 Action 2] F --> G[💾 Action 3] G --> D[🔄 Loop Back] D --> C C -->|Exit| E[✅ End] subgraph Loop["반복 구간"] B F G D end style A fill:#e1f5fe style B fill:#f3e5f5 style F fill:#f3e5f5 style G fill:#f3e5f5 style C fill:#fff3e0 style D fill:#e3f2fd style E fill:#e8f5e8
활용 사례:
  • 대용량 데이터 배치 처리
  • API 페이지네이션 처리
  • 재시도 로직 구현
🎯 패턴 선택 가이드: Sequential은 단순한 작업 체인, Parallel은 독립적 작업 병렬화, Conditional은 상황별 분기, Loop는 반복 처리에 최적화되어 있습니다.

📊 데이터 구조와 참조 방법

n8n에서 데이터를 처리하고 노드 간 참조하는 방법
📋 JSON 기반 데이터
[ { "id": 1, "name": "김철수", "email": "kim@example.com", "metadata": { "created_at": "2024-01-01", "source": "web_form" } } ]
💡 데이터 참조 방식

🔗 직전 노드 참조

// Expression 모드에서 직전 노드의 데이터 참조 {{ $json.name }} {{ $json.email }} {{ $json.items[0].title }}

📍 특정 노드 참조

// 노드명을 지정하여 특정 노드의 데이터 참조 {{ $('Edit Fields5').item.json.name }} {{ $('HTTP Request').item.json.response }} {{ $('Set Data').item.json.user_id }}
💡 핵심 원리:
• 직전 노드: 이미 아이템이 결정된 상태 → ${{json.키}}
• 특정 노드: 노드명으로 지정 필요 → ${{$('노드명').item.json.키}}
• 설정 위치: Fixed가 아닌 Expression 모드에서 사용
📁 데이터 전달 메커니즘: JSON Data(구조화 텍스트)와 Binary Data(파일/이미지)로 구분되어 노드 간 전달됩니다.

🤖 Agentic 노드 - 지능형 처리

인공지능을 활용한 고급 데이터 처리와 분석
🤖 Agent 형
  • LLM이 판단하여 적합한 도구 선택
  • Agent 노드: AI 에이전트 통합 관리
  • OpenAI, Upstage 등 LLM 연동
  • 복잡한 작업 수행을 위한 도구 활용
🔍 Retrieval 형
  • 자연어를 임베딩으로 변환하여 처리
  • Vector Store: 벡터 데이터 저장/검색
  • Document Loader: 문서 내용 추출
  • 텍스트 유사도 기반 정보 검색
📋 AI 에이전틱 워크플로우 노드
💡 에이전틱 워크플로우 특징: AI 모델이 노드에 직접 관여하여 판단·생성 역할을 수행하며, 필요에 따라 추가 도구들과 연계되는 지능형 워크플로우입니다.

🎯 워크플로우 설계 Best Practices
